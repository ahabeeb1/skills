# Prior-Art Research: Grill 2.0 — full-workflow alignment upgrade

**Tier:** Balanced — 5 sub-problems, zero residual unknowns, 3 hard doc-convention constraints; retrofit.
**Date:** 2026-06-09
**Status:** Approved as-is at the Phase 6.4 HITL pivot gate (sub-decisions deferred to grill)
**Internal precedent:** 12 grill records (Tier 0)

## 0. Executive summary

Ship Grill 2.0 as **three markdown-only amendments**: (1) an eighth ambiguity axis — *slice shape* — so the grill interrogates the work breakdown with the same Socratic pressure it applies to feature decisions; (2) a **first-class re-grill edge** on tdd-loop's halt path — a forward step in fresh context with a distilled learning payload, never an in-place spec mutation — plus a fail-fast scope rule for in-flight sibling slices; (3) three **indirect mental-model probes** in grill Phase 1 (premortem, concrete-example demand, door-classification test) that make the user *produce artifacts of understanding* rather than affirm it. Headline trade-off: added per-run ceremony — contained by tier-scaled probe counts and a hard rule that every addition stays a conversation, not a checklist (the documented failure mode of every gate that calcified).

## 1. Problem

socratic-grill drives ambiguity out of *feature decisions* but never validates the slice decomposition, has no sanctioned path back from implementation when reality contradicts the spec, and probes the spec's clarity rather than the user's expectations. All three gaps are confirmed live on v1.24.0, with documented internal incidents.

## 2. Case studies

1. **Internal — 12 grill records (Tier 0).** Slice-shape questions already occur *ad-hoc* in 9 of 12 records and caught a real defect: the v1.20.0 grill's PT-4 exposed a factually wrong slice-ordering justification ("unblocks Spec B" — it didn't; the justification was corrected to "smallest end-to-end tracer-bullet"). All 3 documented mental-model misses (v1.13.0 D4 learnings-ledger cut, v1.19.0 OQ-6 replace-vs-augment correction, v1.22.0 OQ-5/6 dormant-vs-runtime-writer-path) were caught late by user pushback, not by grill. A partial re-entry seam already exists: `NEEDS_CONTEXT → re-dispatch once → BLOCKED("edit-spec-and-redispatch")` (tdd-loop status contract), plus a soft Phase 6 retro question ("re-run socratic-grill if material").
2. **Kubernetes KEP lifecycle (negative example).** Seven statuses, zero backward transitions — and kubernetes/enhancements issue #2960 shows SIGs improvising inconsistent status resets because the process never defined the edge. If the methodology doesn't define the backward path, users invent incompatible ones.
3. **Rust RFC amendments.** In-place edits capped at "very minor"; substantial implementation-discovered changes require a *new* RFC back-linked to the original; a tracking issue absorbs implementation-time detail churn separately. Protects the artifact people built consensus on; gives a clean amendment-vs-supersession boundary.
4. **Example Mapping (Cucumber, Matt Wynne).** 25-minute timeboxed session, four card types (story / rules / examples / questions); forcing *concrete examples per rule* exposes holes in the human's model; red cards inventory the unanswerable; thumb-vote gates readiness. The strongest direct prior art for mental-model validation.
5. **Anthropic (Claude Code best practices + multi-agent research system).** After two failed corrections, restart in fresh context rather than keep patching — accumulated failure pollutes context. In the multi-agent research system, completed sibling work is checkpointed and *salvaged* (resume-from-failure-point), never discarded wholesale.

## 3. Patterns

- **No mature process loops backward in-place.** KEP (undefined → chaos), Rust (new RFC), Anthropic (fresh restart), obra/superpowers `executing-plans` (halt to human, explicit prohibition on autonomous re-plan). Convergent: re-grill is a *forward* step producing a dated, back-linked artifact in fresh context.
- **Conversation + judgment beats checklist contract.** Every working pre-implementation gate is a timeboxed conversation ending in a verdict (Example Mapping vote, Shape Up betting table, premortem debrief); the documented decay mode is hardening into a stage-gate (Scrum.org's Definition-of-Ready anti-pattern; Nick Cameron's RFC-uniformity critique — including *atomization*: if the heavy track hurts, users game slice boundaries to dodge it).
- **Ceremony triggers are ≤5-item falsifiable heuristics.** Google design-docs 3-of-5 questions, Amazon one-way/two-way doors, Shape Up's five betting questions. The trigger's own misclassification is worth probing (a one-way door mislabeled two-way is the expensive failure).
- **Cancellation is cooperative and tiered.** Temporal (graceful cancel requires a poll point; non-polling work only sees termination), GitHub Actions (`fail-fast` default with per-job `continue-on-error` opt-out), sagas (per-step compensation, no global rollback). Completed sibling work is salvage/evidence, not waste.
- **Mental-model probes are indirect.** Premortem ("it failed — why?"; ~30% lift in reason-identification per Mitchell/Russo/Pennington 1989), concrete-example production, deprioritization choice ("which slice would you throw away?" — Humanizing Work's post-split quality test).

## 4. Recommendation

Three amendments, all conversation-shaped, all ADR-0002/0022/0023-compliant:

1. **Slice-shape as the 8th ambiguity axis** (`ambiguity-axes.md`), applied to the spec's slice table as a grillable item class in Phase 1 inventory. Probes: vertical-ness, the deprioritization test, size balance, HITL-gate necessity, ordering justification (the exact v1.20.0 PT-4 catch), parallelizability. Explicitly **not** a new phase, **not** a pass/fail rubric — the Definition-of-Ready anti-pattern is the failure mode to dodge.
2. **First-class re-grill edge.** tdd-loop's halt path gains `suggested_action: "re-grill"` (riding the existing BLOCKED payload — the 4-status vocabulary stays intact). Mechanics: halt → distill a learning payload ("what implementation taught us") → *scoped* grill round in fresh context on the blocked decision only → outcome is either an inline spec patch (Rust's "very minor") or escalation to ADR amendment/supersession, recorded as a new dated grill record back-linked to the original. Sibling scope rule: lead classifies the cause — spec-wide → pause siblings at their next checkpoint; slice-local → siblings run to completion; finished sibling results enter the re-grill payload as evidence.
3. **Three indirect mental-model probes in Phase 1**, tier-scaled in *count* (Quick 1 / Balanced 2 / Deep 3): premortem ("six months out, this failed — what happened?"), concrete-example demand on the riskiest decision, and a door-classification test where grill *challenges* the user's one-way/two-way labeling rather than accepting it. Answers land in a new grill-record section so downstream skills inherit the user's success criteria.

### Steering reconciliation

- **Anchor "peer methodology gates": Honored** — KEP/Rust/superpowers/Shape Up all load-bearing; the two negative examples (KEP's missing edge, RFC uniformity) shape the design as much as the positives.
- **Anchor "requirements-elicitation / shared-mental-model": Honored** — Example Mapping + premortem + door framework are the SP3 backbone.
- **Anchor "agentic re-plan loops": Honored with override** — the anchor implies autonomous re-planning; convergent evidence (superpowers' hard prohibition, Anthropic's restart-fresh rule) says halt-to-human with a structured re-grill. The re-grill edge is human-mediated by design.
- **Look-at obra/superpowers / Kubernetes KEP / RFC processes / internal grill records: all Honored** (12 records mined, not the 5 estimated at invocation).
- **Avoid runtime substrate: Honored** — every mechanism is markdown + existing status vocabulary.

## 5. Decisions to make next (→ grill)

1. Slice-shape placement: 8th axis + Phase 1 item class (recommended) vs a separate Phase 1.5.
2. Re-grill plumbing: new `suggested_action` value (recommended) vs a 5th status — the latter breaks the 4-status vocabulary.
3. The amendment-vs-supersession boundary: what counts as "very minor" for an inline spec patch mid-flight.
4. Sibling-halt default when cause is ambiguous: pause-all (safe) vs run-to-completion (salvage-max).
5. Mental-model probe → grill-record section shape; which downstream skill reads it (write-plan? decision-record?).
6. Whether a re-grill round can itself re-fire the conditional extensions (agent-factors-check/devex-review) or only the named axes.

## 6. Open questions

- Salvage protocol detail: exact format of the "learning payload" a halted slice hands to the re-grill round.
- Does the door-misclassification probe need evidence standards (what would falsify "two-way"?) or stay free-form?
- KEP-style escalating re-review (rising bar per stage) — out of scope for this release; revisit if re-grill rounds become frequent.
