## Patterns (6, including 3 new from addendum)

**A — "Scaffold only what the model can't derive."** Anthropic best-practices doc is explicit; Cognition AI generalizes (give the agent full context, not pre-digested slices). Any phase that re-derives facts Claude would read anyway is ceremony cost.

**B — "Description quality is the primary correctness axis."** Anthropic identifies vague/colliding descriptions as the dominant skill anti-pattern. Chain depth doesn't matter if descriptions don't trigger correctly.

**C — "Error analysis on real traces, not synthetic checklists."** Hamel + Shreya. Synthetic dogfood scenarios approach 100% pass rate (red flag). verify-output is evaluator-first.

**D (addendum) — "Handoff = ownership transfer is a *primitive*, not ceremony."** OpenAI's Agents SDK exposes `Handoff` as one of three top-level primitives. habeebs-skill's `HANDOFF:` lines are not a code smell — same primitive at the markdown layer.

**E (addendum) — "Deterministic workflow scaffolding is a first-class agent type."** Google's ADK explicitly elevates `Workflow Agents` as peers to LLM Agents. habeebs-skill's 6-phase chain IS a workflow agent in Google's taxonomy. Long context does NOT obsolete orchestration structure.

**F (addendum) — "Context engineering is four moves: Write / Select / Compress / Isolate."** LangChain's framework unifies the audit. Every habeebs phase is one or more moves. Coverage assessment: Write ✅✅, Select ✅✅, Isolate ✅, Compress-at-ingest ✅, **Compress-at-overflow ❌ (gap)**.

## Recommendation (5 changes)

### R1 — Audit SYSTEM_CONTEXT contents through Anthropic's prune test.

For each section, apply *"Would removing this cause Claude to make mistakes?"* Drop re-derivable structural facts (Stack / Persistence / Deployment shape / External services / Recent hot files). Keep non-re-derivable cross-session state (Scale envelope / Methodology / Active steering / Last reconciliation / Notable absences / Project mode).

**Land as:** ADR-0010 narrowing ADR-0001 scope. Patch release v1.10.0 — auto-migrates via Phase 0 single-writer.

### R2 — Document HANDOFF lines as navigation hints, NOT state transfer.

This is **already true** (each phase reads previous phase's full output) but docs don't say it explicitly. Add one paragraph to `using-habeebs-skill`. **Land as:** doc patch + ADR-0004 amendment (Part 3 share-full-traces clause, Cognition AI citation).

### R3 — Add real-trace error-analysis cadence.

`chain-postmortem` cadence as section in `using-habeebs-skill` for v1.10.0; promote to skill in v1.11.0 if uptake high. New `docs/agents/postmortems/` directory with transition-failure-matrix template (Hamel/Shreya). Pair with verify-output (which stays as static pre-commit check). **Land as:** ADR-0011 capturing evals philosophy — verify-output (static, pre-commit, KNOWN slop) and postmortems (dynamic, post-incident, NEW categories) are complementary, not peers.

### R4 — Audit description trigger precision.

v1.9.0 trimmed descriptions but didn't measure trigger accuracy. Run one-time audit: ~30 realistic user prompts (15 happy-path + 15 adversarial), measure correct/wrong skill invocations. Anthropic identifies this as THE correctness axis for skills.

**Land as:** `tests/dogfood/13-trigger-precision/` corpus + audit-report.

### R5 (addendum) — Compress-at-overflow protocol.

LangChain's 4-move framework surfaced the gap: habeebs covers Write/Select/Isolate and Compress-at-ingest but has no Compress-at-overflow move. Markdown-only summary-and-flush pattern; 7-section template; passive doc in v1.10.0 with v1.11.0 active-skill promotion criterion (3+ postmortems showing context-distraction).

**Land as:** ADR-0012 + section in `using-habeebs-skill` + `docs/agents/templates/session-summary-template.md`.
