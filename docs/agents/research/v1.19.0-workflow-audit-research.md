# habeebs-skill v1.18.0 — Architecture Audit Memo

**Tier:** Deep
**Date:** 2026-05-25
**Audited surfaces:** ADR/version conflict prevention, methodology folder taxonomy, parallel-dev/worktrees positioning
**Author:** prior-art-research synthesis (Phase 6)

---

## 0. Executive summary

The v1.18.0 advisory-in-flight-reads architecture is canonically correct (no peer prior art exists for the gap it fills; prevention-via-partition is the unanimous industry pattern), but it is bolted onto a methodology-folder layout that has accreted **5 declared subdirs the peer industry universally does not create**, and an ADR/version conflict surface that **5 of 5 reference implementations solve via append-only intent files + release-time aggregation** — a pattern habeebs-skill currently does not use. **Single highest-leverage move:** adopt Changesets-shape append-only intent files for both ADR numbering (slug-only on disk, sequential ID assigned at release) and version bumps (per-PR `.changeset/*.md`, single bump at release), and delete the three dormant/under-populated methodology subdirs (`conflicts/`, `dispatches/`, `grill-records/`) with tombstone ADRs — this collapses the conflict surface to one file class while preserving v1.18.0's earned cross-session detection.

## 1. Problem

Audit habeebs-skill (Claude Code plugin, 18 skills + 20 ADRs, single human author) against production peer practice across three areas: (a) ADR numbering and version-bump conflict prevention, (b) methodology folder taxonomy under `docs/agents/`, (c) parallel-dev and using-worktrees skill positioning. Verdict format is keep/cut/restructure per surface.

## 2. Case studies

- **Backstage (Spotify) — late-binding ADR IDs.** Sequential ADR numbers assigned only at merge-time by maintainer; PRs use a placeholder slug filename. Solves multi-contributor numbering collision without coordination overhead. Highest-signal precedent for habeebs-skill's ADR conflict class. ([backstage/backstage docs](https://backstage.io/docs/architecture-decisions/))
- **Changesets — append-only intent files.** Each PR drops a `.changeset/*.md` declaring intent (bump kind + reason); release CLI aggregates all pending files into one version bump + CHANGELOG. Used by Astro, Storybook, Vercel internal tooling. Single shape solves both version-bump conflicts and changelog-merge conflicts. ([changesets/changesets](https://github.com/changesets/changesets))
- **Cognition (Devin) — "don't build multi-agent systems."** Cognition's June 2025 post explicitly warns against multi-agent fan-out for write tasks; "actions carry implicit decisions" and parallel writers diverge irrecoverably. Anti-pattern citation for any parallel-dev write-task usage. ([cognition.ai/blog/dont-build-multi-agents](https://cognition.ai/blog/dont-build-multi-agents))
- **anthropics/skills repo — zero methodology folders.** Anthropic's own canonical Skills-2.0 repo ships skills as flat `.md` files at the top level with no `docs/agents/`, no ADR directory, no glossary, no triage taxonomy. The reference implementation for the substrate habeebs-skill targets adopts the **opposite** of habeebs-skill's folder strategy. ([anthropics/skills](https://github.com/anthropics/skills))
- **MADR + Fowler — tombstone + immutable path.** Both ADR canons mandate retired records keep their file at the original path with status flipped to `Superseded by ADR-NNNN`; never deleted, never renumbered. Justifies tombstone-ADR strategy when deleting declared-but-dormant directories. ([adr.github.io/madr](https://adr.github.io/madr/), [martinfowler.com/architecture/](https://martinfowler.com/architecture/))
- **Cursor — JSON sidecar per worktree.** Cursor isolates agent state in per-worktree `.cursor/` JSON sidecars, caps concurrency at 8, and treats merge-time conflicts as a human-handed-back event. Validates v1.18.0's sidecar shape AND the prevention-via-partition pattern. ([cursor.com/docs/agent](https://cursor.com/docs/agent))

## 3. Patterns

### Converging (3+ sources, high confidence)

- **A. Append-only intent files + release-time aggregation** (Changesets, release-please, Vercel, Backstage). Same shape solves ADR numbering AND version bumps. **Maps directly to habeebs-skill's two largest conflict classes.**
- **B. Tombstone + forward-pointer + immutable path** (MADR, Fowler, Backstage, Rust RFC). Records never deleted. **Maps to any habeebs-skill restructure that retires declared structure.**
- **C. Worktree-per-agent for write-parallelism** (Claude Code, Cursor, practitioner posts). habeebs-skill already does this via `using-worktrees`.
- **D. Conflicts pushed to merge-time, handed to human** (all 5 AI tools). No peer attempts programmatic resolution. **v1.18.0 is in the same shape.**
- **E. Prevention-via-partition, not detection-and-merge** (Cognition, Cursor, Claude Code, Anthropic). The dominant industry posture.
- **F. Deprecation as structured data with version window** (ESLint `since/until/replacedBy`). Maps to tombstone ADRs with explicit supersession metadata.
- **G. Methodology folders earn existence by file count** (anthropics/skills = 0 folders; Mattpocock = 1 folder with 3 files). **habeebs-skill has 5 subdirs at file-count thresholds peer industry treats as "don't bother yet."**

### Competing (genuine trade-offs)

- **1. ADR ID strategy:** sequential-int (MADR), slug-only (joelparkerhenderson — abandoned sequential), late-binding (Backstage). Date-prefix has **zero** canonical endorsement. Habeebs-skill currently does sequential. Fits scale (single author, low rate) but offers no margin once any second writer joins (already happened de facto in parallel-dev dispatches).
- **2. Multi-agent fan-out:** Cognition (don't, for writes) vs Anthropic (yes, for reads at 15× token cost) vs Cursor (yes, with 8-cap + isolation). parallel-dev falls on **both sides** of this split.
- **3. Worktree config sharing:** git default shares config (footgun); `extensions.worktreeConfig` isolates. habeebs-skill's `using-worktrees` skill does not currently document this hazard.

### Anti-patterns explicitly named in sources

1. Multi-agent write-coordination without isolation (Cognition).
2. Shared `~/.claude.json` for parallel sessions (Claude Code issue #25768, EBUSY bug — validates v1.18.0's sidecar choice).
3. Sequential ADR numbering under parallel writers without a coordinator (implicit in Backstage's switch to late-binding).
4. Merge queues for solo/small-team work (Shopify needed it at 400 commits/day; habeebs-skill is at ~1 commit/day, single author).

## 4. Recommendation for your context

### ADR/version conflict prevention

| Surface | Verdict | Rationale |
|---|---|---|
| **ADR numbering scheme** | **RESTRUCTURE — Backstage late-binding** | PRs land with `adr-<slug>.md` (no number). Release skill assigns the next sequential int + renames + updates the README index at release time. Eliminates parallel-PR numbering collision class entirely. Sequential-int kept for human readability; slug-only (joelparkerhenderson) is the alternative but loses chronological readability the audit-trail benefits from. |
| **plugin.json / marketplace.json version bumps** | **RESTRUCTURE — Changesets-shape `.changeset/*.md` intent files** | Per-PR intent file (`bump: minor`, `why: <line>`). Release skill aggregates all pending changesets, writes single bump to plugin.json + marketplace.json + CHANGELOG, deletes consumed changesets. Pattern A directly. **This is the single most-leverage change in the entire audit** — collapses two file conflict classes into one append-only class. |
| **CHANGELOG.md merge conflicts** | **RESTRUCTURE — same `.changeset/*.md` files** | Aggregation step produces the CHANGELOG entry; no manual edits to CHANGELOG.md mid-flight. Same as version bumps; one mechanism. |
| **v1.18.0 cross-session detection (sidecars + peer-scan + pre-push gate)** | **KEEP AS-IS** | Earned. No peer prior art exists for in-flight session-level read-coordination (E1 from pattern extractor); Claude Code #25768 validates the gap exists; ADR-0019's advisory-in-flight-reads narrowing is the correct ADR-0002 carve-out. Do not strengthen, do not scope down. |

### Methodology folder taxonomy (`docs/agents/`)

| **`specs/`** | 13 | **KEEP** | Earns existence. |
| **`plans/`** | 5 | **KEEP** | Just clears threshold. |

Verified on disk: `issue-tracker.md` has 6 consumers (setup-habeebs-skill, GLOSSARY, SYSTEM_CONTEXT, CLAUDE, AGENTS, 1 spec), `triage-labels.md` has 8 consumers (3 references, setup-habeebs-skill, GLOSSARY, issue-tracker, SYSTEM_CONTEXT, CLAUDE, 1 spec). **Net effect (post-v1.19.0 + consumer-count reconciliation):** `docs/agents/` shrinks from 9 declared subdirs to 7 (`adrs/`, `specs/`, `plans/`, `references/`, `postmortems/`, `research/`, root files). Root files unchanged. Two tombstone ADRs for the cuts (`dispatches/`, `conflicts/`) + one structural fold ADR for `grill-records/` → `specs/`. Cross-references stay valid (Pattern B). Two recommendation reversals since the original synthesis: (1) `research/` promoted across file-count threshold by v1.19.0 PR #42; (2) `issue-tracker.md` + `triage-labels.md` consolidation rejected on consumer-count evidence (6 + 8 consumers, both clear ADR-0009 threshold).

### parallel-dev + using-worktrees positioning

| Skill | Verdict | Rationale |
|---|---|---|
| **`parallel-dev`** | **RESTRUCTURE — bifurcate in skill BODY (post-v1.19.0 scope-narrow)** | Cognition vs Anthropic contradiction (Open Q #1) is real and unresolvable in the abstract. **Resolve by task class explicitly in the skill body:** parallel-dev is APPROVED for read tasks (prior-art-research Phase 4-5 fetchers, audit dispatches) — this is the Anthropic-validated shape at 15× tokens. parallel-dev is RESTRICTED for write tasks (tdd-loop AFK slices) to require (a) per-worktree isolation, (b) ≤8 concurrent dispatches (Cursor cap), (c) explicit independence verification (already required). **Post-v1.19.0 update:** the original recommendation said "split the description with separate trigger language." v1.19.0 added `disable-model-invocation: true` to parallel-dev, so auto-trigger collision is no longer the problem — the bifurcation moves entirely into the skill body as usage guidance + a routing-table note in CLAUDE.md. Skill is well-positioned. **Gap:** does not currently warn about git's `extensions.worktreeConfig` footgun (default shares config across worktrees; one agent's `git config` mutation hits all peers). Add a one-paragraph hazard section. ## 5. Specific decisions to make next

Concrete options. Pick any subset; not a forced spec handoff.

1. **Adopt Changesets-shape intent files for version bumps.** Highest leverage. Edit `release` skill to (a) require a `.changeset/*.md` per release-worthy PR, (b) aggregate at release time, (c) emit the single plugin.json + marketplace.json + CHANGELOG diff. Optionally write an ADR locking the decision; or just adopt the workflow and let the next audit catch it.
2. **Adopt Backstage late-binding for ADR IDs.** Edit `decision-record` skill to write `adr-<slug>.md`; edit `release` skill to assign the next int + rename at release. Worth an ADR (this is a methodology change).
3. **Delete `dispatches/` + `conflicts/` + fold `grill-records/`** (NOT `research/` — promoted by v1.19.0 above the file-count threshold). Write two tombstone ADRs (supersedes 0004 Part 2; supersedes 0018 Part A) and one structural-fold ADR for `grill-records/` → `specs/`. Pattern B keeps cross-references valid.
4. **Bifurcate parallel-dev by task class — in skill BODY only** (post-v1.19.0 scope-narrow). Edit the skill body to add a "Task class" section distinguishing read-task (Anthropic-validated) from write-task (Cognition-restricted) usage with explicit per-worktree-isolation + ≤8-concurrent + independence-verification gates for writes. Add a one-line note to CLAUDE.md routing table. Do NOT edit the description (v1.19.0 just rewrote it). No ADR needed.
5. **Add worktree config-sharing hazard doc to `using-worktrees`.** One paragraph: warn about `git config` mutations leaking across worktrees by default; recommend `extensions.worktreeConfig` opt-in. No ADR; skill-content edit.
6. **Fix SYSTEM_CONTEXT.md ADR-count drift (18 → 20) and investigate single-writer violation.** Either re-run prior-art-research Phase 0, or root-cause whoever last wrote the file outside Phase 0. ADR-0005 is the load-bearing invariant.

## 6. Open questions

Surfaced contradictions and unresolved items. Do not smooth.

1. **Should `parallel-dev` exist at all?** Cognition's anti-pattern applies if it's a write tool; Anthropic's success if read tool. parallel-dev is invoked by prior-art-research Phase 4-5 (READ) AND by tdd-loop AFK slices (WRITE). Recommendation #4 bifurcates rather than picks; whether bifurcation is enough vs scoping to reads-only is unresolved.
2. **Is sequential ADR numbering canonical or obsolete?** joelparkerhenderson (most-starred ADR repo) abandoned sequential; MADR doubled down on it; Backstage late-binds. Recommendation picks late-binding; the underlying canon contradiction remains.
3. **Worktree maturity.** Git's own docs still call worktrees "still experimental"; every peer AI tool treats them as production. habeebs-skill is building on an officially-experimental primitive. No mitigation beyond awareness.
4. **Prevention-via-partition leaves a merge-time human-resolution gap.** All 5 AI tools punt to human at merge. Either this is the correct industry answer to a genuinely undecidable problem, OR it is a collective punt habeebs-skill could improve on. No evidence either way.
5. **SYSTEM_CONTEXT.md ADR-count drift.** File says 18 ADRs; disk has 20. **This is itself an audit finding:** the ADR-0005 single-writer invariant is either being violated, or prior-art-research Phase 0 didn't update on the last two ADR additions. Root-cause needed before trusting SYSTEM_CONTEXT.md as a research input.
