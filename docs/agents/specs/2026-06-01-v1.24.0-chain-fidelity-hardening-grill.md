## Per-item resolutions

### Item 1 — OQ-1 Bundle boundary

- **Starting state:** Spec ships SP2+SP3+SP4+SP1-amendment; spec flagged "grill should pressure-test whether SP3 and SP4 each clear the bar."
- **Axes grilled:** Reversibility (gold-plating risk), Scale (how often does SP4 even fire?)
- **Key questions asked:** SP4 fires only on ADR status→Superseded, which happened ONCE in repo history (ADR-0020). Is a per-PR lint for a once-a-quarter event gold-plating?
- **Resolution:** DECIDED
- **Decision:** Keep all four. SP4 is cheap (a small bash scan gated on `Superseded` status) and catches the exact error the chain relied on a human getting right last week. Low cost, real-if-rare catch. The rarity is a feature — the check is near-free when no supersession is present.
- **Spec update:** None (spec already specifies all four).

### Item 2 — OQ-2 SP2 banned-phrase precision

- **Starting state:** Spec's candidate set `NOT renamed | no longer | used to | replaces the old | now that .* changed | set in v[0-9]`, explicitly unmeasured against the 19 live SKILL.md bodies.
- **Axes grilled:** Failure modes (false positives), Observability (measure don't theorize — the "boring backup" probe applied: just run it).
- **Key questions asked:** What is the ACTUAL false-positive count on the live corpus? (Ran the scan during the grill.)
- **Measurement result:** 4 hits, **all 4 false positives** — `decision-record:156` ("Deprecated — no longer current"), `systematic-debugging:17` ("a test that used to pass now fails"), `using-habeebs-skill:127` ("no longer maps to reality"), `using-worktrees:389` ("a path that no longer exists"). Zero true archaeology. The naive set is 100% FP.
- **Resolution:** DECIDED
- **Decision:** Redesign to target **self-referential archaeology** — sentences narrating the skill/methodology's OWN past behavior. Banned shapes require a self/skill subject: `this skill used to`, `previously (this|we)`, `was renamed`, `replaces the old <X>`, `(in|set in) vN this changed`, `formerly`, `no release step <verb>s it`, `<X> are NOT renamed`. Drop bare `no longer` / `used to` (proven too broad). The discriminator is the sentence's *subject* (the methodology's evolution), not the verb phrase. Narrowed regex must corpus-test to zero (the v1.23.0 cleanup already removed the real offenders).
- **Spec update:** Slice 1 acceptance criteria — replace the banned-phrase list with the self-referential set; add the corpus-test-result note (4 FP on the naive set → narrowed).

## New decisions surfaced during grilling

1. **The archaeology discriminator is the sentence subject, not the verb phrase.** Real archaeology narrates the *methodology's own evolution* ("this skill used to write X"); a false positive describes a *domain scenario in present tense* ("a test that used to pass now fails"). SP2's regex must encode subject-awareness (self/skill subject required), not phrase-matching. This is the load-bearing insight from the corpus measurement and belongs in the ADR.
2. **Corpus-first is not just rollout hygiene — it's regex *design* input.** Running the scan before writing the lint changed the lint's fundamental shape (broad→narrow). The SP5 discipline should state: corpus-test BEFORE finalizing the regex, not just before enabling the block.
