## Per-item resolutions

### OQ1 — Slice-shape placement — DECIDED
**Axes:** Reversibility, Migration.
**Resolution:** 8th ambiguity axis in `ambiguity-axes.md` + a Phase 1 rule that the spec's slice table is a standing inventory item class. No separate Phase 1.5 — a dedicated phase is the Definition-of-Ready calcification path; the item-class rule already guarantees the slice table is never skipped while keeping probes judgment-applied.

### OQ2 — Re-grill plumbing — DECIDED
**Axes:** Reversibility (one-way door), Migration.
**Resolution:** `suggested_action: "re-grill"` riding the existing BLOCKED payload. No consumer needs status-level visibility; a 5th status would break the locked 4-status vocabulary across tdd-loop, parallel-dev, verify-output, and all dispatch records for zero migration benefit.

### OQ3 — Amendment-vs-supersession boundary — DECIDED
**Axes:** Failure modes, Reversibility.
**Resolution:** Falsifiable blast-radius rule. A mid-flight spec patch is MINOR iff it (a) changes no other slice's acceptance criteria, (b) adds no slice, and (c) touches neither the Architecture section nor Concrete picks. Anything else is SUBSTANTIAL → ADR amendment/supersession, recorded in the back-linked re-grill record.

### OQ4 — Sibling-halt default — DECIDED
**Axes:** Concurrency, Failure modes.
**Resolution:** Pause-all default. When the halt cause is ambiguous, in-flight siblings pause at their next checkpoint (worktree isolation makes the pause lossless); the lead may explicitly classify the cause slice-local to let siblings run to completion. Finished sibling results always enter the re-grill payload as evidence.

### OQ5 — Mental-model record consumers — DECIDED
**Axes:** Observability.
**Resolution:** New "User mental model" grill-record section (success criteria, door classifications + recorded undo costs, premortem risks) with TWO mandatory consumers: `write-plan` reads success criteria as acceptance-gate candidates; `decision-record` reads door labels into ADR consequences. Record-only (no consumer) was rejected as dead ceremony.

### OQ6 — Extension re-fire in scoped rounds — DECIDED
**Axes:** Concurrency (scope), Reversibility.
**Resolution:** Domain-touch rule. A scoped re-grill round re-fires a conditional extension only when the blocked decision is inside that extension's domain, applied to that single item. Never a full proactive sweep (contradicts scoping); never-fire was rejected (misses domain gaps in the changed decision).

### OQ7 + DX-1 — Learning payload + halt UX — DECIDED
**Axes:** Failure modes, Observability; devex-review first-error-quality lens.
**Resolution:** 7-field payload: `blocked_slice`, `blocked_decision` (quoted from spec), `expected_vs_observed`, `evidence` (test output / file refs), `attempted_resolutions`, `scope_classification`, `salvaged_sibling_results`. Halt surfaces as a fixed-format gate block (mirroring the research pivot-gate UX): what blocked, payload summary, and exactly two exits — inline spec patch (minor) or ADR escalation (substantial).

### OQ8 — Door-classification evidence — DECIDED
**Axes:** Reversibility.
**Resolution:** One-follow-up rule. Every "two-way" label gets exactly one concrete probe — "what's the undo cost, concretely?" — and the answer is recorded in the User mental model section. No formal evidence standard (ceremony disproportionate to a per-decision label); no free-form acceptance (misclassification is the documented expensive failure).
