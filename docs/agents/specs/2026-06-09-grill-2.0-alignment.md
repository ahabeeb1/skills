# Implementation Spec: Grill 2.0 — full-workflow alignment upgrade

**Slug:** `grill-2.0-alignment`
**Status:** Grilled
**Version:** v1.25.0 (candidate — assigned at release per changeset convention)
**Tier:** Balanced (inherited from research)
**Spec'd from:** [docs/agents/research/2026-06-09-grill-2.0-alignment-research.md](../research/2026-06-09-grill-2.0-alignment-research.md)
**Spec'd on:** 2026-06-09

## TL;DR

Three markdown-only amendments that make `socratic-grill` align user and agent across the whole workflow: a slice-shape ambiguity axis, a first-class re-grill edge on tdd-loop's halt path (with a sibling fail-fast scope rule in parallel-dev), and three tier-scaled indirect mental-model probes. 5 slices, all AFK, mostly sequential (shared file surfaces).

## Architecture

No runtime components — the "architecture" is a contract change across three skill surfaces. `socratic-grill` gains one axis, three probes, and a scoped re-entry mode; `tdd-loop` gains a `re-grill` suggested action on its existing BLOCKED status plus a learning-payload format; `parallel-dev` gains a halt-scope classification rule. Data flow: tdd-loop halt → learning payload → scoped grill round (fresh context) → spec patch or ADR-amendment escalation → resume.

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Slice validation placement | 8th ambiguity axis + Phase 1 item class | Conversation-shaped; avoids the Definition-of-Ready checklist anti-pattern |
| Re-grill direction | Forward step, fresh context, dated back-linked record | Convergent across KEP (negative), Rust RFC, Anthropic restart-fresh |
| Re-grill plumbing | `suggested_action: "re-grill"` on BLOCKED | Preserves the 4-status vocabulary |
| Autonomy boundary | Halt-to-human, structured re-grill — never autonomous re-plan | superpowers prohibition + Anthropic two-corrections rule |
| Sibling handling | Lead classifies spec-wide vs slice-local; salvage finished work as evidence | GitHub Actions fail-fast knob + Anthropic salvage pattern |
| Mental-model probes | Premortem, concrete-example demand, door-classification challenge | Indirect probes outperform direct "do you understand?" |
| Probe scaling | Count scales by tier: Quick 1 / Balanced 2 / Deep 3 | Ceremony scales, decision quality doesn't (tier invariant) |

## Trade-offs accepted

- Added per-run grill ceremony (more questions per run) in exchange for catching slice defects and expectation mismatches pre-implementation.
- Re-grill is human-mediated — AFK throughput pays a human round-trip on every spec-invalidating ambiguity.
- Slice-shape grilling stays judgment-based — no mechanical pass/fail, so quality depends on the conversation.

**Description:** Grill interrogates the spec's slice table with a new axis: vertical-ness, deprioritization test, size balance, HITL-gate necessity, ordering justification, parallelizability. A grill run on a spec with a planted horizontal decomposition now surfaces it.

**Blocked by:** None

### Slice 3 — Re-grill edge end-to-end (AFK)

**Description:** tdd-loop's BLOCKED payload gains `suggested_action: "re-grill"` plus a learning-payload format; socratic-grill gains a scoped re-entry mode (input: payload + blocked decision; output: inline spec patch for minor, ADR-amendment escalation for substantial; always a new dated grill record back-linked to the original).

**Acceptance criteria:**
- [ ] `tdd-loop/SKILL.md` status contract lists `re-grill` as a BLOCKED suggested action with the halt mechanics; Phase 6 retro question upgraded to reference it.
- [ ] Learning payload has the 7 fields: blocked_slice, blocked_decision (quoted from spec), expected_vs_observed, evidence, attempted_resolutions, scope_classification, salvaged_sibling_results.
- [ ] Halt surfaces as a fixed-format gate block: what blocked, payload summary, exactly two exits (inline patch | ADR escalation).
- [ ] `socratic-grill/SKILL.md` defines the scoped round: fresh-context framing, named-decision scope, the blast-radius boundary (minor = no other slice's ACs change, no new slice, Architecture/Concrete-picks untouched; else substantial → ADR), dated back-linked record naming, and the domain-touch rule for extension re-fire.
- [ ] Dogfood: simulated mid-slice ambiguity fixture → halt emits well-formed payload → re-grill round produces a record with back-link and a spec patch.

**Test strategy:** Dogfood scenario — at `tests/dogfood/<next-free-N>-regrill-edge/` (confirm N at implementation).

**Blocked by:** #2 (same file: `socratic-grill/SKILL.md`)

## Dependency DAG

```
1 → 2 → 3 → 4 → 5
```

Strictly sequential — Slices 1-3 share `socratic-grill/SKILL.md` and the output template; 4 consumes 3's vocabulary; 5 documents everything. 0% parallelizable: this is feature shape (all changes target one skill's surface), not a missed-dependency artifact.

## Parallelization

None — see DAG note. Do not dispatch via `parallel-dev`; run `tdd-loop` per slice in order.

## Revisit triggers

- Re-grill rounds fire >2× in a single release cycle → revisit whether the amendment-vs-supersession boundary is too strict (forcing ceremony) or too loose (spec drift).
- A slice-shape grill item produces a false positive that the user overrides in 3+ consecutive runs → re-tune the axis probes.
- Mental-model probes start getting one-word answers (probe fatigue) → reduce Balanced count to 1, re-evaluate phrasing.
