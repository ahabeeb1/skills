## Trade-offs accepted

- **Phase 0 first-message context shrinks.** Phase 1 of prior-art-research used to skip "what's your stack?" because Phase 0 had it cached. Now that section is gone — Claude greps `package.json` on the fly instead. Cost: one extra read; benefit: aligned with Anthropic's prune test.
- **Postmortem cadence is aspirational, not enforced.** OSS-plugin reality: real-user traces are scarce. The skill provides the template + protocol; uptake depends on user discipline. Hamel's ~100/cycle minimum is unreachable here.
- **Trigger-precision audit is one-shot.** Recurring measurement would require either CI infrastructure (rejected per ADR-0002) or quarterly manual passes (process burden). v1.10.0 ships the one-time audit; recurring cadence is a v1.11.0+ revisit candidate.
- **Compress-at-overflow is *guidance*, not automation.** The harness already auto-compacts; this protocol covers gaps *between* harness compactions (e.g., during a manual loop or long tdd-loop run). Cost: relies on user noticing context pressure; benefit: ADR-0002 preserved.
- **No vendor benchmarks for Google ADK validation.** Our "Google validates the chain shape" claim rests on Google's architectural endorsement, not reliability numbers (Google publishes none for ADK).

## Revisit triggers

Scale or capability conditions that mean the spec needs to be re-evaluated:

- **Q1 promotion — postmortem section → skill.** 10+ real postmortems land in 90 days OR explicit user request for a `/postmortem` skill → promote section in `using-habeebs-skill` to standalone `chain-postmortem` skill with description tuned to failure-shaped trigger phrases. Revisit at v1.11.0.
- **Q4 promotion — Compress-at-overflow passive → active.** 3+ postmortems show context-distraction as the failure mode → upgrade R5 passive-doc to an active skill that detects overflow signals (conversation length thresholds, prompt-cache TTL pressure). Revisit at v1.11.0 / v1.12.0.
- **R3 cadence enforcement.** If real-trace volume reaches >20 postmortems/quarter, the manual cadence may merit CI tooling (would require ADR-0002 amendment). Revisit at v1.12.0.
- **R4 corpus drift.** If 5+ skills land between v1.10.0 and v1.11.0, the trigger-precision corpus needs re-curation. Revisit at v1.11.0 or 6 months, whichever first.
- **Vendor consensus shift.** If Anthropic publishes Skills 2.0 v2 that changes description-driven invocation semantics, OR if Google ADK retires Workflow Agents as a first-class type, revisit chain-shape validation.
- **Long-context erosion of Compress need.** If model context windows grow to ≥10M and prompt-cache TTL extends past 1hr, R5 protocol may become moot. Revisit at v2.0.0 or on model-family upgrade.
