# Implementation Spec: gstack Capability Adoption

**Slug:** `gstack-capability-adoption`
**Status:** In-progress
**Spec'd from:** Prior-art-research "Adopting capabilities from gstack into habeebs-skill" (2026-05-17, Deep mode, 19 sources)
**Spec'd on:** 2026-05-17
**Grilled on:** 2026-05-18 — see `v1.13.0-gstack-capability-adoption-grill.md`

## TL;DR

Adopt three substrate-free capabilities from garrytan/gstack into habeebs-skill, re-implemented as pure markdown per Pattern A (idea-port, not skill-port): a new `security-audit` skill, a new `release` skill closing the chain after `tdd-loop`, and a `devex-review` conditional grill extension. **Staggered release: v1.13.0 ships `security-audit` alone; v1.14.0 ships `release` + `devex-review`.** Portfolio ceiling: 3 new skills, 0 new conventions. ADR-0002 stands unamended — gstack's runtime-coupled half is rejected, and the research confirms that is the correct call.

## Architecture

Three new markdown skill files slot into the existing 15-skill plugin. `security-audit` is a standalone slash-invokable skill. `release` is a new terminal chain link after `tdd-loop`, folding in doc-sync, and amends the ADR-0003 commit-block hook so release tag-pushes are no longer blocked on the default branch. `devex-review` is a conditional extension of `socratic-grill`, mirroring `agent-factors-check`. No runtime substrate is introduced; every capability is a prompt driving Bash/Read/Edit/Grep.

```
existing chain:  research → spec → grill → record → plan → tdd-loop
                                    │                          │
                          devex-review (cond.)          release (NEW terminal link)
                                                               │
standalone:  security-audit (slash-invoked, /security-audit)
```

## Concrete picks (from research + grill)

| Decision | Choice | Reason |
|---|---|---|
| Security review gap | New `security-audit` skill, ported from gstack `/cso` | habeebs-skill has no security review; `verify-output` explicitly disclaims it. |
| `security-audit` trigger (D1) | Standalone slash skill only | Security review isn't every-PR work; matches gstack `/cso`; no new chain phase/predicate. |
| `security-audit` depth (D1) | Trimmed core | Markdown-only; drop trend-tracking (persisted state), CI/CD scans, JSON artifacts. |
| Release gap | New `release` skill after `tdd-loop` | Chain ends at `tdd-loop`; release is manual. |
| Release tag-push hook (D2) | Fix the hook (amend ADR-0003) | The commit-block hook wrongly blocks `git push <tag>` on main; refine the matcher. |
| Doc-sync (D5) | Fold gstack `/document-release` into `release` | Coverage-map + CHANGELOG sell-test; doesn't merit its own skill. |
| DX lens (D3) | `devex-review` conditional extension of `socratic-grill` | habeebs-skill is itself a developer tool; `agent-factors-check` is the proven pattern. |
| `devex-review` trigger (D3) | Developer-facing tools (CLI / SDK / library API / plugin / framework) | Mirrors `agent-factors-check`'s domain gate. |
| Learnings-ledger (D4) | Cut — not a real gap | Covered by ADRs + `postmortems/` + SYSTEM_CONTEXT reconciliation log; a manual ledger decays. |
| Release sequencing (D6) | Stagger: v1.13.0 = `security-audit`; v1.14.0 = `release` + `devex-review` | Isolates the hook/ADR change from the new skills. |
| Runtime-coupled gstack features | Reject all (browser, `/qa`, `/canary`, `/codex`, GBrain, `/land-and-deploy`) | ADR-0002; no UI surface, no deploy target, no retrieval-scale problem. |
| ADR-0002 | Stands unamended | No compelling revisit candidate — recorded as a finding. |

## Trade-offs accepted

- No browser-driven QA or visual/design review — habeebs-skill has no UI surface.
- No cross-model (Codex) review — only in-model adversarial passes.
- No deploy/canary/benchmark automation — `release` stops at PR + tag, never touches production.
- No cross-session learnings ledger — by design; learnings live in ADRs / `postmortems/` / SYSTEM_CONTEXT reconciliation log.
- Chain grows from 15 to 18 skills — the portfolio ceiling (3 skills, 0 conventions) is the mitigation.

## Vertical slices

Numbered in dependency order. Each slice cuts end-to-end: a complete skill with SKILL.md, references, a dogfood test scenario, and wiring into the discovery surfaces (`.claude-plugin/plugin.json`, `README.md`, `skills/using-habeebs-skill/SKILL.md` chain map). HITL = human-in-the-loop required; AFK = autonomous-friendly.

### Slice 1 — `security-audit` skill (HITL) — target v1.13.0

**Description:** A new standalone slash-invokable skill (`/security-audit`) running a substrate-free static security audit and emitting a markdown report. Re-implemented from gstack `/cso`, trimmed to markdown-only.

**Acceptance criteria:**
- [ ] `skills/security-audit/SKILL.md` exists; description under the 1,536-char cap; trigger-precision passes the description-budget dogfood check.
- [ ] Methodology lives in `skills/security-audit/references/` and covers exactly the trimmed core: attack-surface census, secrets archaeology (git history), OWASP Top 10, STRIDE per-component, confidence-gated false-positive filter.
- [ ] No cross-run trend-tracking/fingerprinting, no CI/CD/infra scans, no JSON artifacts — markdown report only.
- [ ] Invoking `/security-audit` on a sample repo produces a markdown report with confidence-scored findings; zero runtime-substrate dependency (Bash/Read/Grep only).
- [ ] Wired into `plugin.json`, `README.md`, and the `using-habeebs-skill` chain map as a standalone primitive.

**Test strategy:** Integration (dogfood scenario) — at `tests/dogfood/17-security-audit/` — adversarial scenarios with planted vulnerabilities the audit MUST surface, plus a no-vuln control it MUST pass clean (mirrors the `09-category-critic` 4-scenario pattern).

**Blocked by:** None

**Notes:** HITL — the methodology trim and description trigger-precision tuning are taste calls. Ships as v1.13.0 on its own.

### Slice 2 — `release` skill + ADR-0003 hook amendment (HITL) — target v1.14.0

**Description:** A new terminal chain link invoked after `tdd-loop`. Assembles a release: version bump across `plugin.json` + `marketplace.json`, CHANGELOG entry, clean commit history, PR body, a doc-sync sub-phase (coverage-map of shipped features lacking docs + CHANGELOG "sell test"), and a tag-push. Also amends the ADR-0003 commit-block hook.

**Acceptance criteria:**
- [ ] `skills/release/SKILL.md` exists; description under cap; trigger-precision passes.
- [ ] Running the skill produces a version-bumped, CHANGELOG-updated, clean-history branch with a PR body — no deploy/canary/benchmark steps.
- [ ] Doc-sync sub-phase folded in: maps shipped features lacking docs; applies the CHANGELOG "sell test" (entry explains value, not just the change).
- [ ] `hooks/preventing-commits-to-default.sh` refined so a tag-only push (`git push origin <tag>`, `git push --tags`) is distinguished from a branch-commit push and allowed on the default branch.
- [ ] ADR-0003 amended in place to document the tag-push carve-out; the PreToolUse hook dogfood scenario updated to cover it.
- [ ] Wired into `plugin.json`, `README.md`, and the `using-habeebs-skill` chain map as the terminal chain link after `tdd-loop`.

**Test strategy:** Integration (dogfood scenario) — at `tests/dogfood/18-release/` — verifies version-bump correctness, CHANGELOG insertion, and the now-allowed tag-push path; a control verifying no deploy actions are emitted. Plus the updated hook dogfood scenario.

**Blocked by:** None (functionally independent of Slice 1)

**Notes:** HITL — touching the ADR-0003 PreToolUse hook is a real architectural seam. Ships in the v1.14.0 bundle.

### Slice 3 — `devex-review` conditional grill extension (AFK) — target v1.14.0

**Description:** A conditional extension of `socratic-grill` adding a developer-experience review lens (onboarding friction, first-time-developer roleplay, API/CLI ergonomics) when the spec under grill is a developer-facing tool. Mirrors `agent-factors-check`'s structure.

**Acceptance criteria:**
- [ ] `skills/devex-review/SKILL.md` exists; description under cap; structured as a conditional grill extension, not a standalone chain phase.
- [ ] Trigger predicate: fires only when the spec's product is consumed primarily by developers — CLI, SDK, library API, plugin, or developer framework.
- [ ] Produces one Socratic question per DX gap, feeding the grilling agenda; returns review-lens output, not code changes; conforms to the grill handoff contract.
- [ ] Wired into `plugin.json`, `README.md`, and `socratic-grill`'s conditional-extension references alongside `agent-factors-check`.
- [ ] A one-paragraph note added to `using-habeebs-skill`: cross-session learnings live in ADRs / `postmortems/` / SYSTEM_CONTEXT reconciliation log by design — no separate ledger.

**Test strategy:** Integration (dogfood scenario) — at `tests/dogfood/19-devex-review/` — a developer-tool spec that MUST trigger the lens and a non-developer-facing spec that MUST NOT (trigger-precision control).

**Blocked by:** None

**Notes:** AFK — `agent-factors-check` is a near-exact template. Ships in the v1.14.0 bundle.

## Parallelization

Slices #2 and #3 (the v1.14.0 bundle) are functionally independent but both edit the shared discovery surfaces — `.claude-plugin/plugin.json`, `README.md`, `skills/using-habeebs-skill/SKILL.md`.

- **Recommended:** sequential — #2 then #3 — within the v1.14.0 cycle. The portfolio is small; sequencing avoids merge-conflict churn on the shared surfaces.
- **If parallelized:** dispatch #2 and #3 to separate worktrees via `using-worktrees`, then serialize the discovery-surface edits in a final integration commit.

## Revisit triggers

- habeebs-skill ships a UI/dashboard surface → the gstack browser engine becomes a real ADR-0002 revisit candidate.
- A destructive-command incident appears in a `postmortems/` entry → reconsider a `/careful`-style guard (deferred per Q1).
- 3+ chain runs that should have been killed before research as not-worth-building → reconsider a demand-validation gate (deferred per Q3).
- A third independent skill collection surfaces the same missing capability twice → treat as a confirmed gap regardless of this spec.
