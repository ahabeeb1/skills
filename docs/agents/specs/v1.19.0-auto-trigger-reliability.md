# Implementation Spec: Auto-trigger reliability (v1.19.0)

**Slug:** `v1.19.0-auto-trigger-reliability`
**Status:** Locked (ADR-0007 amendment recorded 2026-05-24)
**Tier:** Balanced (inherited from research)
**Spec'd from:** `docs/agents/research/2026-05-24-auto-trigger-reliability.md`
**Spec'd on:** 2026-05-24

## TL;DR

Rewrite all 18 skill descriptions to the converged trigger-first / literal-quote / directive-imperative template, demote **11** chain-internal skills via `disable-model-invocation: true`, and add a routing table to CLAUDE.md so the agent has explicit task-type → skill mapping. 8 vertical slices, mixed HITL/AFK, sequenced through an ADR-0007 amendment and a real-session eval that augments (then sunsets) the gameable synthetic dogfood corpus.

## Success metric

Post-release at +30 days, the transcript eval (Slice 6) measures: **(sessions with any of `"build"`, `"add"`, `"refactor"`, `"fix this"`, `"design"`, `"implement"` in the user prompt AND the matched entry-point skill fires within 2 turns) / (sessions with any of those keywords)**. v1.19.0 ships green if the post-release ratio exceeds the pre-release baseline by **>10 percentage points**. Below that threshold, file v1.20.0 candidate with the `You MUST use this skill when…` (imperative-with-pronoun) variant — NOT a revert.

## Architecture

Three coordinated layers, all markdown/frontmatter — no runtime substrate (preserves ADR-0002):

1. **Description anatomy** — every SKILL.md description follows `[Capability ≤8 words]. [Imperative trigger directive] when [literal user phrase 1], [phrase 2], or [phrase 3]. [Tight scoped anti-trigger].` Cap drops to 1024 chars (Anthropic spec). Target avg drops to 300.
2. **Auto-invocation scope** — only true entry points stay auto-invocable. Chain-internal skills get `disable-model-invocation: true`; they remain `/slash-invocable` and HANDOFF-invocable.
3. **Routing primer** — `## Skill routing` block at the top of `CLAUDE.md` gives the model an explicit task-type → skill table the always-loaded prompt sees before fuzzy-match runs.

```
USER PROMPT
   │
   ├── CLAUDE.md routing table  ────► explicit task-type→skill (Layer 3, wins on a match)
   │
   ├── fuzzy-match across descriptions
   │     │
   │     ├── 8 auto-invocable skills (Layer 1 rewritten descriptions)
   │     └── 10 chain-internal: SKIPPED (Layer 2 disable-model-invocation)
   │
   └── falls through to direct task execution if nothing matches
```

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Description anatomy | Trigger-first + literal-user-phrase-in-quotes + directive-imperative | 650-trials study: directive imperative 94-100% activation vs 37-87% for advisory `Use when` (Cohen's h = 1.83, p < 0.0001). Anthropic's `pdf`/`docx`/`claude-api` and Superpowers all converged on the same pattern. |
| Description length | 150-400 chars target, ≤1024 hard cap | Anthropic actual cap is 1024 (not 1200 — ADR-0007 used outdated number). Superpowers averages 8-30 words; Mattpocock ~40-80 words. |
| Chain-internal skill auto-invoke | Disabled via `disable-model-invocation: true` | Precedent: Mattpocock `zoom-out`. 10 of 18 habeebs skills only run because upstream HANDOFF or explicit `/slash` — they dilute the trigger pool without value. |
| Auto-invocable skill set | 5 entry points + 3 support meta | `prior-art-research`, `systematic-debugging`, `deep-modules`, `security-audit`, `parallel-dev` as entry; `using-habeebs-skill`, `setup-habeebs-skill`, `using-worktrees` as support context. |
| Trigger directive intensity | `ALWAYS use` / `You MUST use` for the 3 highest-leverage entry points; `Use when` for primitives | Match Anthropic `claude-api`'s `TRIGGER when:` directness for high-catchment entries; preserve agent judgment for primitives like `deep-modules` to avoid over-firing. |
| Routing primer location | Inline in `CLAUDE.md` (not split into separate file) | CLAUDE.md is always-loaded; one indirection cuts half of session reads. Existing `## Agent skills` block already establishes the convention. |
| ADR-0007 update mechanism | In-place amendment (not ADR-0019 supersession) | The cap-and-budget framework stays; only the pushy-trigger preservation rule gets replaced. Pattern shift is empirical refinement, not architectural change. |
| Eval mechanism | Real-session transcript scraping, not synthetic corpus | Existing dogfood 13 reported 34/34 (100%) but user observes failures in real sessions — Hamel Husain's "synthetic-prompts-approach-100%-by-construction" red flag is confirmed empirically. |
| Renaming to gerund? | **HOLD** | Slash-command surface (`/research`, `/grill`) is load-bearing muscle memory. If Layer 1 alone doesn't move firing-rate, revisit in v1.19.0. |

## Trade-offs accepted

- **Per-trigger keyword density drops sharply.** Today's 528-717 char descriptions enumerate 5-10 trigger phrases each. The 150-400 char target preserves only 2-4 highest-signal trigger phrases per skill. The bet is that 3 well-chosen literal user phrases beat 8 paraphrased ones (consistent with the 650-trials Variant B finding: keyword accumulation alone doesn't fix passive framing).
- **`disable-model-invocation` removes 10 skills from `/help`-style auto-discovery surfaces.** The user can still `/spec`, `/grill`, etc. but a fresh user who's never seen habeebs-skill before won't see those skills mentioned by Claude unprompted. Mitigation: the `using-habeebs-skill` skill (still auto-invocable as context) introduces the full chain on the first relevant trigger.
- **Routing table in CLAUDE.md adds ~25 lines of always-loaded budget.** Roughly 250-400 tokens per turn. Justified by the firing-rate improvement; revisit if context budgets tighten.
- **Eval is real-session-only.** No script that simulates 30 prompts and asserts which skill fires (per Hamel — premature). Instead the eval is a manual transcript-review cadence after the v1.19.0 release. Slower feedback, but unfakeable.
- **Description rewrites are a one-way door for v1.19.0.** If post-release firing-rate is worse, rollback means re-editing 18 files. Reversible, but tedious. Mitigated by Slice 1 grilling decisions before bulk rewrite.

### Slice 3 — Rewrite 18 SKILL.md descriptions to the converged template (HITL)

**Description:** Rewrite every SKILL.md `description:` field to follow the v1.19.0 anatomy: `[Capability ≤8 words]. [Use when | ALWAYS use | You MUST use] [literal user trigger 1], [trigger 2], or [trigger 3]. **Concrete rewrites for the 3 highest-leverage skills** (from research §Recommendation Layer 1):

### Slice 6 — Real-session transcript eval cadence (HITL)

**Description:** Define and document the v1.19.0+ trigger-firing eval as a quarterly transcript review. The new eval *augments* (not replaces) the dogfood 13 synthetic-corpus per OQ-6 resolution — dogfood 13 becomes a "regression baseline" re-run after Slice 3, while the transcript eval becomes the *primary* firing-rate signal. After 2 quarters of dual-tracking (v1.20.0+), if real-eval consistently surfaces failures dogfood 13 misses, sunset dogfood 13.

Write a new doc at `docs/agents/references/trigger-firing-eval.md` that codifies:

1. **Source:** `~/.claude/projects/<repo-slug>/sessions/*.jsonl`
2. **Sample:** last 30 sessions OR last 14 days, whichever yields more transcripts. Pin sample-set in a date-range header so re-runs are reproducible (idempotency requirement per OQ-6).
3. **Anonymization:** redact each user prompt to a keyword-level summary (preserves trigger-language structure without leaking content).
4. **Scoring rubric:** the success-metric numerator/denominator defined at the top of this spec — sessions containing trigger keywords (`"build"`, `"add"`, `"refactor"`, `"fix this"`, `"design"`, `"implement"`) where the matched entry-point skill fired within 2 turns, divided by all sessions containing those keywords. Plus per-skill recall (did each entry-point skill fire when it should have?).
5. **Cadence:** first week of each quarter, OR 7 days after any v1.X release, whichever sooner.
6. **Failure response:** if post-release lift <10pp, file a v1.X+1 description-tuning issue with the imperative-with-pronoun variant (`You MUST use this skill when…`) — NOT a revert.

Also: update dogfood 13's README to mark the corpus as "**regression baseline** — synthetic-prompt precision check; primary firing-rate signal lives in `docs/agents/references/trigger-firing-eval.md` from v1.19.0."

## Revisit triggers

Conditions under which this spec needs re-evaluation:

- **Layer 1 doesn't hit the 10pp lift threshold.** If the +30-day post-release transcript eval (Slice 6) shows <10pp improvement against pre-release baseline, the next variant is **`You MUST use this skill when…`** (imperative-with-pronoun), shipped as v1.19.0. NOT a revert to current phrasing — that would lose the directionality data. Three variants on the imperative gradient (advisory `Make sure to` → directive `ALWAYS use` → directive-with-pronoun `You MUST use`) gives us 3 data points instead of 2.
- **CLAUDE.md grows past 200 lines.** Per OQ-5 resolution, split the `## Skill routing` block into `docs/agents/SKILL_ROUTING.md` and replace inline content with one-line reference. 200 lines is Anthropic's recommended system-prompt density ceiling.
- **Anthropic ships a description-cap change.** If the 1024 cap moves up or down, Slice 4 dogfood constant has to follow.
- **New skill added without `disable-model-invocation` decision.** Slice 7's `setup-habeebs-skill` update should make this self-correcting, but if a future v1.X skill lands and the auto-invoke decision is left implicit, revisit the 7-skill auto-invocable list.
- **Claude Code adds explicit trigger telemetry.** If a future Claude Code build exposes skill-firing events as a hook or log, replace the manual transcript-review cadence (Slice 6) with the telemetry source.
- **Gerund-renaming becomes load-bearing.** If 2 consecutive quarters of transcript eval show persistent undertriggering on 3+ skills despite Layer 1 + v1.19.0 fixes, the next step is gerund-renaming (deferred from this release). That would be v1.20.0 candidate.
- **Dogfood 13 sunset trigger.** At v1.20.0 (2 quarters post-v1.19.0), if real-session eval consistently surfaces failures dogfood 13 misses, retire dogfood 13. Until then, dogfood 13 runs as regression baseline.
- **A-7/A-8/A-9 implicit ambiguities** (rollback shape, /help interaction with `disable-model-invocation`, Layer 1/2/3 cutover ordering) were explicitly skipped at grill time. Re-open if implementation surfaces concrete pain in any of: a partial-Layer rollback, a fresh user reporting they can't find demoted slash commands, or a session where Slice 5's routing table contradicts Slice 2's demoted set.
