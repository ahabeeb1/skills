## Architecture

Three coordinated layers, all in-repo markdown/JSON, all consumed by the existing `release` skill at merge time (extending v1.19.0's `## Description-policy audit` section — NOT a rewrite):

1. **`.changeset/<slug>.md` intent files.** Per-PR markdown drop carrying `bump: patch|minor|major` + `why: <line>` frontmatter. Feature branches NEVER edit `plugin.json` / `marketplace.json` / `CHANGELOG.md`. `release` skill aggregates pending changesets at merge, writes the single bump + CHANGELOG entry, deletes consumed changesets. Pattern A directly (Changesets / release-please).
2. **Late-binding ADR IDs.** `decision-record` writes `docs/agents/adrs/adr-<slug>.md` (no number). `release` skill at merge: (a) scans for any `adr-<slug>.md` files, (b) reads `adrs/README.md` for the next sequential int, (c) renames each to `NNNN-<slug>.md`, (d) regenerates README index. Backstage pattern directly.
3. **Folder cuts + tombstones.** `dispatches/` and `conflicts/` deleted (0 files; declared but never used). `grill-records/` content moved into `specs/<name>-grill.md` naming + directory deleted. ADR-0004 Part 2 + ADR-0018 Part A amended in place with `Status: Partially superseded by ADR-NNNN` lines. One NEW ADR records the structural-fold + supersession decision. Pattern B (tombstone + immutable path) preserved — retired directories get nothing left behind because they're empty; cross-references in 18 SKILL.md files + AGENTS.md + CLAUDE.md updated where they exist.

```
[PR opens]                                    [release skill runs at merge]
    ↓                                                    ↓
.changeset/abc.md  (bump+why)             scan .changeset/*.md → aggregate → write
                                                    plugin.json + marketplace.json
adrs/adr-foo.md   (no number)                  +  CHANGELOG.md entry
                                          scan adrs/adr-*.md → assign next int →
                                                    rename + update README
```

## Trade-offs accepted

- **Two-phase release workflow.** Every release-worthy PR must include a `.changeset/<slug>.md` file. This is a contributor-discipline shift; `release` skill enforces it via the existing v1.19.0 `## Description-policy audit` step (extended). Mitigated by clear CLI ergonomics in `decision-record` + `release` skills.
- **Bot-equivalent at merge time.** habeebs-skill has no CI; the "release skill" runs locally and is the single coordinator. If the user merges a PR without running `release` skill afterward, changesets pile up. Acceptable because (a) the very next release picks them up, (b) `release` skill already runs the dogfood audit step that fails loud if state is wrong.
- **`decision-record` writes `adr-<slug>.md` lose chronological sortability until the rename runs.** Mitigated: branch is short-lived; rename happens at merge; pre-merge sortability is unaffected (the slug carries the topic, the date carries the chronology via git).
- **Cannot rename existing ADRs even if we wanted to.** Asymmetric ADR scheme (old: `NNNN-slug.md`, new: same final shape but went through `adr-<slug>.md` intermediate state). Mitigated by single-shape final form: at rest, all ADRs are `NNNN-slug.md`; only the *creation path* changed. Readers see no difference.
- **Three directory deletions.** `dispatches/`, `conflicts/`, `grill-records/` are removed from `docs/agents/`. AGENTS.md + CLAUDE.md mention `dispatches/` in their "Agent skills" section (verified during research) and require text updates. SKILL.md files in `parallel-dev` + `cross-session-detect` may reference these paths internally; will update during slice work.

### Slice 3 — Release skill: changeset aggregation + path audit (HITL)

**Description:** Extend the existing `release` skill SKILL.md with **two new phases**. (a) "Changeset aggregation" — runs as the FIRST step of release PR creation. Reads `.changeset/*.md`, picks highest bump (major > minor > patch), writes `plugin.json` + `marketplace.json` + CHANGELOG.md entry aggregating all `why:` lines as bullet points, deletes consumed changesets, commits atomically. (b) "Path audit" — runs at PR-creation time on every PR. Reads `git diff --name-only main...HEAD`, applies required/optional/never matrix, halts loud if a required path was modified without a `.changeset/*.md`.

**Acceptance criteria:**
- [ ] `release` skill SKILL.md gains **"Changeset aggregation" phase** with explicit ordering as the **first step of release PR creation**: aggregate → bump → CHANGELOG entry → delete changesets → commit (atomic, all-or-nothing) → push branch → open PR. Tag-push step (post-merge) unchanged from v1.19.0.
- [ ] `release` skill SKILL.md gains **"Path audit" phase** at PR-creation time with explicit matrix:
  - **REQUIRED:** `skills/`, `hooks/`, `.claude-plugin/`, `plugin.json`, `marketplace.json`
  - **OPTIONAL (emits INFO note, does not block):** `docs/`, `CLAUDE.md`, `AGENTS.md`, `README.md`, `CHANGELOG.md`
  - **NEVER required:** `tests/`, `.gitignore`, `.github/`, `.gitattributes`
- [ ] Helper script at `skills/release/scripts/aggregate-changesets.sh`: reads all `.changeset/*.md`, prints aggregated bump + reasons, applies via temp-staging-dir (atomic-or-rollback per OQ-2), deletes changesets. **Exit codes: 0 success, 1 aborted clean (no state change), 2 aborted dirty (manual intervention).** **Supports `--dry-run` and `--help`.**
- [ ] Helper script at `skills/release/scripts/check-changeset-required.sh`: implements the path-audit matrix. Exit codes: 0 audit pass, 1 required-without-changeset halt. **Supports `--dry-run` and `--help`.**
- [ ] Helper integrates with existing v1.19.0 `## Description-policy audit` section (extend, do not duplicate)
- [ ] **5 named error messages** with explicit format, asserted by dogfood scenario 25:

**Test strategy:** Integration test — at `tests/dogfood/23-changeset-aggregation/` and `tests/dogfood/25-changeset-required-check/`. Uses fixture worktrees to verify the full read → bump → write → delete flow + path-audit matrix + error messages.

**Blocked by:** #2

**Notes:** HITL because integration with v1.19.0's release skill ordering is non-trivial. **Two-PR version-slot race accepted** per Pattern D (audit memo): two simultaneous release PRs bumping the same version slot will collide at git merge time; second merge fails loudly on CHANGELOG + plugin.json conflict; operator resolves by closing the second PR + re-running aggregation.
