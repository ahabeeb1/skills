## Per-item resolutions

### OQ-1 — Same-day collision rule

- **Starting state:** Spec offered halt-loud / `-N` suffix / sub-day timestamp.
- **Axes grilled:** Concurrency (two ADRs same day, parallel sessions), Failure modes (what does the author see), Reversibility.
- **Key questions asked:** Same-day is fine because the slug disambiguates — but what about same date AND same slug (identical filename)? Is a duplicate slug a signal the name is too vague?
- **Resolution:** DECIDED.
- **Decision:** **Halt loud on a true duplicate.** `decision-record` refuses to write if `YYYY-MM-DD-<slug>.md` already exists and asks for a more specific slug. No silent overwrite, no magic suffix. Rationale: a duplicate slug on the same day means the slug isn't descriptive enough — the fix is a better name, not a counter. Matches log4brains/Shopware (slug uniqueness IS the contract).
- **Spec update:** Slice 1 acceptance criterion already covers "same-day collisions handled per OQ-1 ruling" — annotate that the ruling is halt-loud.

### OQ-2 — Cross-reference style

- **Starting state:** Spec offered `[[YYYY-MM-DD-slug]]` wikilink / short date-ID / title+link.
- **Axes grilled:** Reversibility (one-way-door: once refs proliferate, restyling is costly), DevEx (cross-ref ergonomics — `ADR-0007` is 8 chars; a dated slug is ~50).
- **Key questions asked:** Is typing a full dated slug in prose an ergonomic regression? How do new ADRs cite the 24 frozen integer ones?
- **Resolution:** DECIDED.
- **Decision:** **Title + markdown link in prose** ("see the [dated-naming decision](../adrs/2026-05-28-….md)"). Frozen integer ADRs continue to be cited as `ADR-0007`. No brittle `[[slug]]` strings to mistype; the link survives even if a human shortens the prose title. Bidirectional convention locked: new→new = title+link; new→old = `ADR-00NN` (the integer IS still that file's stable ID); old→new (rare, if a frozen ADR is amended) = title+link.
- **Spec update:** Add to Slice 6 / the ADR: document this cross-ref convention in `adrs/README.md` cutover note.

### OQ-3 — Scope + version traceability

- **Starting state:** Spec OQ-3 (ADRs-only vs full sweep); Slice 4 conditional.
- **Axes grilled:** Migration (does spec→plan→release traceability survive losing the version from the filename), Scale (3 more skills touched).
- **Key questions asked:** If specs/plans go dated, does the version that leaves the filename survive in frontmatter, and does the spec→plan→release link still resolve?
- **Resolution:** DECIDED.
- **Decision:** **Full sweep.** ADRs + specs + plans + grill-records all go dated. The version identifier moves into a frontmatter `Version:`/`Release:` field so spec→plan→release traceability is preserved by frontmatter, not filename. **Slice 4 ships** (not cut).
- **Spec update:** Slice 4 stays in; remove the "if grill resolves OQ-3 as ADRs-only, this slice is cut" conditional from its Notes — replace with the confirmed full-sweep ruling.

### OQ-4 — Cutover marker

- **Starting state:** README note / tombstone ADR / frontmatter `scheme:` field.
- **Axes grilled:** Observability (how a reader knows the cutover happened), DevEx (first-time reader opens the dir, sees mixed schemes).
- **Key questions asked:** Where does a human or Codex/Cursor reader look first, and what's the lowest-ceremony signal?
- **Resolution:** DECIDED.
- **Decision:** **A section note at the top of `adrs/README.md`** explaining "0001-0024 are frozen integer ADRs; 2026-05-28 onward are dated `YYYY-MM-DD-<slug>.md`." Lives where readers already look; no new file; no per-file frontmatter redundancy.
- **Spec update:** Slice 5 acceptance criterion already references the marker per OQ-4 — annotate as README-note.

### OQ-5 — README index upkeep

- **Starting state:** New `update-adr-index.sh` vs hand-maintained.
- **Axes grilled:** Failure modes (drift between dir and index), Migration (the deleted script used to do this), the "boring backup" probe.
- **Key questions asked:** For a low-volume solo repo with no CI, is a script worth the maintenance surface vs decision-record appending one markdown row?
- **Resolution:** DECIDED.
- **Decision:** **Hand-maintained by `decision-record`** — it appends the index row when it writes the ADR (one markdown row). No new script. Matches the no-substrate ethos and the low write-volume. (If volume ever grows or drift recurs, a regen script is the revisit.)
- **Spec update:** Slice 5 — replace the "new script vs hand-maintained" optionality with the hand-maintained ruling; the slice is now mostly the README cutover note + decision-record's index-append instruction.

### OQ-6 — ADR-0020 disposition

- **Starting state:** In-place partial-supersession note vs full new superseding ADR.
- **Axes grilled:** Reversibility, Failure modes (risk of implying the Changesets half was superseded too).
- **Key questions asked:** ADR-0020 has two independent halves — how do we kill one and keep the other without confusing future readers?
- **Resolution:** DECIDED.
- **Decision:** **Full new superseding ADR.** The new dated ADR supersedes ADR-0020 wholesale, then **explicitly re-states the Changesets-shape version-bump half as still-in-force** (so the supersession is not read as killing changesets). ADR-0020's status becomes "Superseded by [dated ADR]" with a one-line forward note clarifying the changesets mechanism continues under the new ADR.
- **Spec update:** **Slice 6 changed** — was "partial-supersession note on ADR-0020"; now "full new superseding ADR that re-states the Changesets half as retained." Update Slice 6 description + acceptance criteria.

### OQ-7 — Causal-ordering sufficiency (open question)

- **Starting state:** Open question, not a decision: is supersession-by-forward-link strong enough that "ADR-7 came before ADR-12" is never missed?
- **Axes grilled:** Reversibility, the "what would have to be true" probe.
- **Key questions asked:** Dates lack the monotonic "came-after" that integers gave for free — when does that bite?
- **Resolution:** DEFERRED with trigger.
- **Revisit trigger:** If a reader or agent ever misattributes decision order because dated filenames lack monotonicity, adopt a hybrid: dated filename + a monotonic `seq:` frontmatter field. Until then, the Title+link convention (OQ-2) plus explicit forward-links carry causal ordering. (Already listed in the spec's Revisit triggers.)
- **Spec update:** None — already a revisit trigger.

## New decisions surfaced during grilling

1. **Bidirectional cross-ref convention is asymmetric and that's fine** — new ADRs cite old ones by their still-valid integer ID (`ADR-0007`), and old/new ADRs cite new ones by title+link. The integer IDs of 0001-0024 remain permanently valid identifiers; freezing the *naming scheme* does not invalidate the *references*.
2. **In-flight-branch migration (DevEx).** Any branch that created an `adr-<slug>.md` under the old late-binding scheme before this ships must rename it to `YYYY-MM-DD-<slug>.md` before merge (decision-record no longer produces the unnumbered form). One-time, well-bounded (single author, short-lived branches). The superseding ADR's migration note should call this out, mirroring ADR-0020's own in-flight-branch note.
3. **dogfood-28 error-message clarity (DevEx).** When the dated-string lint fires, its message must distinguish "dated string in prose body (banned)" from "dated filename/frontmatter (allowed)" so a contributor isn't confused by the carve-out. Fold into Slice 3 acceptance criteria.
