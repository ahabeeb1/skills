Whether new specs/plans switch to `YYYY-MM-DD-<slug>.md` is **Open Question 3** below — `/grill` resolves it before Slice 4 implements. Do not read this filename as having pre-decided OQ-3.

## TL;DR

We stop binding ADR identity to the release event. `decision-record` will write ADRs (and, pending OQ-3, specs/plans/grill-records) with their **final `YYYY-MM-DD-<slug>.md` name at creation time**, where the **slug — not the bare date — is the uniqueness key**. The release-driven late-binding rename machinery is deleted; the 24 existing integer-named ADRs (0001-0024) are frozen in place with zero cross-reference edits. Six slices in three phases; the Changesets-shape version-bump machinery (the *other* half of ADR-0020) is untouched throughout.

## Architecture

A naming-convention + writer-relocation change across markdown docs and shell scripts — no runtime substrate (ADR-0002 preserved). Today the write path is asymmetric: `decision-record` writes `adr-<slug>.md` (no ID) → the `release` skill's Phase 3.5 invokes `assign-adr-ids.sh` to scan, assign the next integer, rename to `NNNN-<slug>.md`, and append the README index. After this change the write path is direct: `decision-record` writes `YYYY-MM-DD-<slug>.md` at creation, no release step touches ADRs, and the README index is maintained without the rename script. The directory permanently mixes two schemes — `0001-0024` (frozen integers) + `YYYY-MM-DD-<slug>` (everything new) — which is shipped precedent (Rails 2.1).

```
BEFORE (release-coupled):
  decision-record ──writes──> adr-<slug>.md ──[release Phase 3.5: assign-adr-ids.sh]──> NNNN-<slug>.md + README row

AFTER (creation-time, release-independent):
  decision-record ──writes──> YYYY-MM-DD-<slug>.md (+ README row)        release skill: no ADR step
                              slug = uniqueness key; date = sort order
  legacy 0001-0024 ───────── frozen, never renamed, all ADR-00NN refs stay valid
  .changeset/* + aggregate-changesets.sh ───────────────────── UNTOUCHED (ADR-0020 half #2 survives)
```

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Identity binding | At **creation** (decision-record), not at release | Fixes "not everyone has releases"; the core complaint |
| New artifact name | `YYYY-MM-DD-<slug>.md` | Sorts chronologically, legible; matches Superpowers (plans/specs), log4brains/Shopware (ADRs), and existing `research/` dir |
| Uniqueness key | The **slug**, not the bare date | Bare `YYYY-MM-DD` collides same-day (MADR #28 verbatim); log4brains/Shopware use slug-as-key |
| Existing 24 ADRs | **Freeze** integer names, zero cross-ref edits | ~1,241 `ADR-00NN` refs across ~117 files stay valid; freeze-old/date-new is shipped precedent |
| Scope | ADRs + specs + plans + grill-records (specs/plans pending OQ-3) | User Phase-1 choice: full sweep of dated-able artifacts |
| Late-binding machinery | **Delete** mechanism #1; **keep** mechanism #2 (Changesets) | Research §5: clean separation — share coordinator, not code/state |
| ADR-0020 disposition | **Partial supersession** (late-binding-ID half only) | The version-bump half is independent and stays in force |
| dogfood-28 self-consistency | **Carve-out**, not contradiction | Filename-date (immutable identifier) ≠ prose-date (decaying version-archaeology) |

## Trade-offs accepted

- **Causal ordering degrades.** A monotonic integer says "came after" for free; a date is fuzzy (same-day = no intrinsic order) and the slug says nothing. Supersession must lean entirely on explicit forward-links. (The one genuine regression — OQ-1/OQ-7.)
- **Mixed-scheme directory under plain `ls`.** Integers `0001-0024` sort lexically *before* `2026-*`, so dated files cluster after all legacy integers — chronologically slightly off but acceptable; no sorting tooling in a markdown repo.
- **Asymmetric creation path persists** (just inverted): old ADRs were born numbered-at-release; new ones are born dated-at-creation. At rest both are immutable filenames; the asymmetry is only in how they were created.
- **No forge-number option.** The objectively-cleanest decouple (Rust "number-at-merge = PR#") is rejected because this project has no CI and merges manually — hand-copying PR numbers is worse than dates+slug.

## Open questions — RESOLVED by grill (2026-05-28)

All resolved in [`v1.23.0-dated-artifact-naming-grill.md`](./v1.23.0-dated-artifact-naming-grill.md):

- [x] **OQ-1 — Same-day uniqueness rule.** DECIDED: slug is the key; `decision-record` **halts loud** if `YYYY-MM-DD-<slug>.md` already exists (true duplicate = slug too vague → demand a more specific slug). No suffix, no overwrite.
- [x] **OQ-2 — Cross-reference style.** DECIDED: **title + markdown link** in prose for new ADRs; frozen integer ADRs keep `ADR-00NN`. No `[[slug]]` wikilinks. Bidirectional: new→new = title+link, new→old = `ADR-00NN`.
- [x] **OQ-3 — Scope + version traceability.** DECIDED: **full sweep** — ADRs + specs + plans + grill-records all dated; version moves to frontmatter `Version:`/`Release:`. **Slice 4 ships.**
- [x] **OQ-4 — Cutover marker.** DECIDED: a **section note at the top of `adrs/README.md`** ("0001-0024 frozen integers; dated thereafter").
- [x] **OQ-5 — README index upkeep.** DECIDED: **hand-maintained by `decision-record`** (appends one index row at write time). No script.
- [x] **OQ-6 — ADR-0020 disposition.** DECIDED: **full new superseding ADR** that re-states the Changesets-shape version-bump half as still-in-force. ADR-0020 status → "Superseded by [dated ADR]".
- [ ] **OQ-7 — Causal-ordering sufficiency.** DEFERRED with trigger: if decision order is ever misattributed because dated filenames lack monotonicity, adopt a hybrid dated-filename + monotonic `seq:` frontmatter field. (See Revisit triggers.)

## Revisit triggers

- **A same-day same-slug collision actually occurs in practice** — validates or invalidates the OQ-1 disambiguation choice; if the chosen mechanism is awkward, revisit (e.g. add sub-day timestamp like Rails `HHMMSS`).
- **Causal-ordering loss bites** — if a reader/agent ever misattributes decision order because dates lack monotonicity, reconsider a hybrid (dated filename + monotonic frontmatter `seq:`).
- **A second author joins** — the parallel-session collision pressure that motivated dates increases; re-confirm slug-as-key holds, or adopt forge-number-at-merge once CI exists.
- **Anthropic ships a first-party ADR convention for Claude Code** — audit dated-naming against it and migrate if warranted (carried over from ADR-0020's revisit triggers).
- **The mixed-scheme directory confuses multi-tool readers** (SP4 left this unverified) — if Codex/Cursor/OpenCode mis-parse the dir, add stronger cutover signaling.
