# Implementation Spec: v1.5.0 Environment Binding (minimal)

**Slug:** `v1.5.0-environment-binding`
**Status:** Grilled
**Spec'd from:** prior-art-research output (in-conversation, 2026-05-11) + socratic-grill outcome that cut the project-mode field
**Spec'd on:** 2026-05-11

## TL;DR

Make `SYSTEM_CONTEXT.md` load-bearing by giving 6 chain skills a halt-if-missing rule that redirects to `setup-habeebs-skill`. Capture the choice as ADR-0001 — the repo's first dogfooded decision record. That's the whole release. Two slices.

## Architecture

`SYSTEM_CONTEXT.md` is already written by `prior-art-research` Phase 0 and read by every chain skill on init. Today, missing the file means "use defaults silently" — which means most users never run setup, which means triage labels / issue tracker / domain glossary stay unconfigured. v1.5.0 changes one rule: the 6 chain skills halt with a clear redirect when the file is missing.

```
[user invokes any chain skill]
        ↓
        is docs/agents/SYSTEM_CONTEXT.md present?
        ├── yes → proceed
        └── no  → halt: "SETUP REQUIRED — invoke /groundwork or /research first"
```

Engineering primitives (`parallel-dev`, `deep-modules`, `vertical-slice`, `using-worktrees`, `systematic-debugging`) do NOT halt. They're invoked from inside the chain (already gated) or standalone (debugging — halting hurts more than helps).

## Concrete picks (from research + grill)

| Decision | Choice | Reason |
|---|---|---|
| Environment-binding primitive | `docs/agents/SYSTEM_CONTEXT.md` (existing) | Audit vs tmux + claude-mem confirmed: in-repo markdown is portable, reviewable, multi-runtime. No change. |
| Halt severity | Mandatory on 6 chain skills | Suggested = status quo. Mandatory + a clean redirect message keeps the bar low. |
| Halt scope | 6 chain skills only | Primitives can still run standalone. `prior-art-research` Phase 0 is exempt (it's the writer). |
| Project-mode field | **Rejected** | Chain already detects brownfield-ness imperatively from existing artifacts (ADRs, code, tests). A declarative mode field is a second source of truth that decays. No precedent in Anthropic / Superpowers / mattpocock / OMC. |
| `setup-habeebs-skill` UX | Verify each of the 3 sections has a one-keystroke default-accept path | Friction balance for the new mandatory bootstrap. Small wording edits, no structural change. |

## Trade-offs accepted

- **First-install friction.** Halt fires on first chain invocation in a fresh repo. Mitigated by the redirect being unambiguous and `setup-habeebs-skill` defaults being one-keystroke accept.
- **No semantic memory.** Cross-session recall stays grep + ADRs. claude-mem / memsearch remain optional add-ons.
- **No upfront project context detection.** Per Anthropic's "iterative validation" guidance, the chain detects state on demand from artifacts. No project-mode field.

## Open questions

None remaining. Grill resolved Q1–Q7; the spec cut absorbed Q2–Q5 (no mode field means no detection heuristic, no refresh trigger, no corruption fallback for the field). Q6 (concurrency invariant) documented inline in slice #1. Q7 (v1.4.x migration) is a no-op since no schema change.

### Slice 1 — Halt-if-missing rule on 6 chain skills + setup UX polish (`AFK:full-auto`)
