## TL;DR

Three highest-leverage parallel-subagent additions plus four prereq fixes that close existing contract gaps. Touches `prior-art-research`, `tdd-loop`, `write-plan`, and `parallel-dev` SKILL files; introduces three new agent prompt files; no new ADR-grade architectural moves beyond the durable-dispatch-state question. **Eight vertical slices**, four of which parallelize, target release **v1.7.0** (hooks already owns v1.6.0).

## Architecture

The chain gains a coverage critic upstream and an end-to-end implementation dispatcher downstream. `prior-art-research` Phase 2 fans out a category-completeness critic before search burns budget. `write-plan`'s `pgroup-N` labels — today inert — become the trigger that `tdd-loop` reads when entering implementation: groups of size ≥2 get dispatched concurrently via `parallel-dev`, each subagent running its own red-green-refactor in its own git worktree and returning a Superpowers-style 4-status code. The dispatcher (`parallel-dev`) gains a concurrency soft-cap, a documented hypothesis-probe sub-pattern, and the actual agent prompt files its own SKILL.md already references.

```
prior-art-research:
   Phase 2 (decompose) ─→ Phase 2.5 NEW: category-completeness critic ─→ Phase 4 (search)

write-plan output:
   pgroup-N table ─┐
                   ↓
tdd-loop entry:    detects pgroup, calls parallel-dev
                                          ↓
                       parallel-dev: dispatches N subagents (cap=5)
                            each: worktree + own TDD cycle
                            each returns: DONE | DONE_WITH_CONCERNS | BLOCKED | NEEDS_CONTEXT
                                          ↓
                       parallel-dev: aggregates, sequential merge to main
```

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Coverage failure fix | Category-completeness critic between Phase 2 and Phase 4 | LangGraph pattern; bounded iteration cap; directly attacks the missed-category failure mode that drove this research |
| Implementation dispatch | Worktree-per-worker + sequential merge to main | Pattern H (Superpowers, appxlab); avoids concurrent-merge hell; soft cap 5 from appxlab's empirical ceiling |
| Subagent return contract | Superpowers' 4-status: `DONE | DONE_WITH_CONCERNS | BLOCKED | NEEDS_CONTEXT` | Machine-readable signal beats arbitrary text; no fork — adopt verbatim |
| Hypothesis-probe sub-pattern | AlphaCode generate-N-filter-to-K | Documented in `parallel-dev` for future `systematic-debugging` Phase 3.5 fan-out (v1.8.0+) |
| Synthesis pattern | Lead-agent synthesis with dedicated `synthesizer` subagent in fresh context | Anthropic multi-agent research; avoids lead context exhaustion |
| Convergence on critics | Dedupe-and-cluster (not vote) | Divergent Socratic / coverage questions all have value |
| Release scope | Recommendations 1, 8, 9 only | Three is one cohesive story; surfaces 6, 10, 11, 12, 13 defer to v1.8.0+ |

## Trade-offs accepted

- **Token cost amplification.** Phase 2.5 critic adds 1 extra subagent call per research run; Phase 6 CitationAgent (deferred to v1.8.0) would add 1 more. Anthropic reports 15× cost for full multi-agent fan-out; this release adds an estimated 1.2-1.4× to research runs and 1.0× (no change) to implementation runs that don't have pgroups.
- **Synchronous blocking.** Subagent dispatch is synchronous (Anthropic's pattern). A slow subagent blocks the dispatcher. Mitigated by the concurrency soft-cap (5) keeping fan-out bounded.
- **Durable dispatch state introduces a new in-repo artifact category.** `docs/agents/dispatches/<dispatch-id>.json` is borderline ADR-0002: in-repo (compliant) but runtime-generated and not authored (borderline). Treated as a write-only audit log, not a substrate; cleared on each chain run. ADR-0004 will formalize.
- **`docs/agents/dispatches/<dispatch-id>.json` is the only file written by subagents in shared scope.** Single-writer invariant (dispatcher owns final write) preserved by having subagents return SHAs and the dispatcher write the record.
- **No automatic re-fan-out in v1.7.0.** Phase 2.5 critic produces a list of missing categories; the lead agent decides whether to add them. Bounded at 1 iteration. Re-fan-out automation (Anthropic CitationAgent pattern) deferred to v1.8.0.
- **No prompt-based hooks involved.** This release is skill-file-only; no new `hooks/` files. The hooks framework from v1.6.0 is unaffected.
- **No async approval queue or stateful suspend/resume.** Git is the durability layer; idempotent re-invocation of `tdd-loop` is the resume mechanic. Mid-flight uncommitted subagent work is lost on kill (user resolves manually). F6 satisfied via simplicity — added during grill (Item 9).

- Dogfood scenario 07 (Phase 2.5 critic) silently passes too easily across 3+ different feature types — the critic is rubber-stamping, not catching real gaps. Revisit the critic prompt and add adversarial decomposition tests.

- ADR-0002 amendment (ADR-0004) is rejected by the grill phase — dispatch records can't live in `docs/agents/dispatches/`. Revisit alternatives (gitignored `.dispatches/`, per-worktree state, in-memory only).
