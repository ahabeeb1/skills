## TL;DR

Rename `docs/agents/CONTEXT.md` → `docs/agents/GLOSSARY.md` everywhere (writer template + 2 readers + setup's emitted `## Agent skills` block), make `setup-habeebs-skill` chain into `prior-art-research` Phase 0 inline as a new final phase, and fork halt copy so GLOSSARY-missing redirects to `/setup` while SYSTEM_CONTEXT-missing redirects to `/research`. 3 vertical slices, single brownfield session. Closes ADR-0001's latent bug (the ADR specified setup should defer the recon write to Phase 0 "when invoked through the chain"; setup never implemented that chain). No migration shim — personal repo.

## Architecture

Two context files at `docs/agents/`, split by **writer lifecycle**, not by topic:

- `GLOSSARY.md` — human-authored, domain-stable. Setup writes the template; user fills in concepts over time. Read by `deep-modules` (vocabulary) and `draft-spec` (slice naming).
- `SYSTEM_CONTEXT.md` — tool-derived, environment-bound. `prior-art-research` Phase 0 is the single writer (ADR-0001 invariant preserved). Read by 5 downstream chain skills + parallel-dev preamble.

`setup-habeebs-skill` becomes the one-invocation bootstrap: writes the human-answered config (GLOSSARY template + issue-tracker.md + triage-labels.md + adrs/README + `## Agent skills` block), then **invokes** `prior-art-research` Phase 0 so SYSTEM_CONTEXT.md exists by the time setup returns. Phase 0 retains exclusive write ownership — setup orchestrates, Phase 0 writes.

```
/setup invocation
   ├── Phase 1   look around (existing files)
   ├── Phase 2   ask Section A: issue tracker
   ├── Phase 3   ask Section B: triage label vocabulary
   ├── Phase 4   ask Section C: domain doc layout
   ├── Phase 5   write GLOSSARY.md, issue-tracker.md, triage-labels.md, adrs/README.md
   ├── Phase 6   append `## Agent skills` block to AGENTS.md / CLAUDE.md
   ├── Phase 7   *NEW* — invoke prior-art-research Phase 0 reconnaissance
   │              (Phase 0 cache-check: if SYSTEM_CONTEXT.md exists+fresh, skip)
   └── Phase 8   confirm (lists all 5 files written, including SYSTEM_CONTEXT.md)
```

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Separation principle | Two files, split by writer lifecycle (writer + refresh cadence) | DDD/Diátaxis/Nygard precedent; matches the accidental split's load-bearing axis |
| Rename target | `CONTEXT.md` → `GLOSSARY.md` | Filename now encodes lifecycle role; closes joelparkerhenderson collision shape |
| `SYSTEM_CONTEXT.md` | Unchanged | 5 readers + 4 ADRs reference it; moving it costs more than it earns |
| Setup chains into Phase 0? | Yes — as a new Phase 7 | Honors ADR-0001's stated intent; single-invocation bootstrap; ADR-0001 single-writer invariant preserved (setup invokes; Phase 0 writes) |
| Phase 0 idempotency from setup | Skip-if-fresh per existing Phase 0 cache-check logic | Re-running /setup on a configured repo is a no-op for SYSTEM_CONTEXT |
| Halt copy | Fork by missing file: GLOSSARY → "/setup"; SYSTEM_CONTEXT → "/research" | Two writers = two halt paths |
| GLOSSARY template content | Commented prompts (`<!-- list domain concepts here -->`), no interactive Q&A | Deterministic across harnesses (Claude Code + Codex + Cursor + OpenCode) |
| Migration shim | None | Personal repo; manual `git mv` if any other repo needs it |
| Phase 0 mid-setup failure | No rollback; setup writes are independently idempotent | Re-running /setup re-detects existing files and only does missing work |

## Trade-offs accepted

- **Two files, not one.** Readers will look in two places. Acceptable: writers and refresh schedules are genuinely different.
- **No auto-bootstrap in *foreign* tooling.** A user invoking `/research` directly (not via `/setup`) still creates SYSTEM_CONTEXT.md but not GLOSSARY.md. We don't auto-create GLOSSARY from `/research` — that'd violate the two-writer separation.
- **One-time prose churn across 4-5 skill files** to do the rename. Cost is bounded; benefit is permanent filename-disambiguation.
- **No migration shim.** Repos pinned to v1.7 that have CONTEXT.md keep working only if their consumer skills also stay on v1.7. Acceptable — personal repo, manual `git mv` is trivial when needed.
- **ADR-0001 supersede.** ADR-0001's "Notable absence: No `docs/agents/CONTEXT.md`" line becomes outdated when slice 3 dogfoods. ADR-0005 (to be written via `/record`) supersedes the relevant paragraphs.
