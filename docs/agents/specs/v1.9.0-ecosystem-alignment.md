## TL;DR

v1.9.0 closes the audit gaps surfaced by the ecosystem alignment research: trim every SKILL.md description ~25% (~500-800 tokens/turn recovered), drop the unrecognized `next-skills` frontmatter field, ship a new `verify-output` skill that runs between `tdd-loop` GREEN and commit, document the `/abort-chain` convention in `using-habeebs-skill`, retrofit GLOSSARY.md consumption into the three middle-chain skills, and extract the SYSTEM_CONTEXT mtime-check into a shared reference doc. **Six vertical slices; description-trim lands first; the four post-trim slices parallelize.**

## Architecture

Six in-repo refactor / convention slices. Zero new runtime substrate. Zero new dependencies. Touches: every `skills/*/SKILL.md` frontmatter (slices 1+2), `using-habeebs-skill/SKILL.md` body (slice 4), three middle-chain SKILL.md Phase 0 sections (slice 5), 10 SKILL.md Phase 0 sections + one new reference file (slice 6), one new `skills/verify-output/` directory (slice 3).

**New directory convention introduced (grill outcome):** `docs/agents/references/` becomes the home for chain-shared cross-cutting helpers (the mtime-check protocol from slice 6 is the first inhabitant). Skill-specific helpers remain under `skills/<name>/references/`. Rule of thumb: used by 3+ skills → shared `docs/agents/references/`; otherwise local.

```
(descriptions)   (next-skills)                          │

                   │                                    │

```

## Concrete picks (from research)

| Decision | Choice | Reason |
|---|---|---|
| Description-trim target | ~600 chars/description avg; 1,200 char hard ceiling | Current avg 785, max 946; trim recovers ~500-800 tokens/turn without losing trigger pushiness |
| Trim mechanics | Move "Inspired by X" credits to SKILL.md body; tighten "Do NOT use" lists | Credits add tokens with zero trigger value; "Do NOT use" lists often verbose past the point of diminishing return |
| `next-skills` frontmatter | Remove from all 14 skills; convert to `## See also` body section where non-obvious | Unrecognized by Claude Code harness — silently ignored. Body section costs nothing per-turn (lazy-loaded) |
| `verify-output` skill | New skill; runs between `tdd-loop` GREEN and commit; ADR-0004 4-status contract | Closes OMC `ai-slop-cleaner` / `ultraqa` gap without runtime coupling |
| `/abort-chain` UX | Documented sub-section in `using-habeebs-skill` — NOT a full new skill | Lightweight; abort is rare enough that a body convention beats a dedicated skill |
| GLOSSARY.md consumers | Retrofit `draft-spec`, `socratic-grill`, `write-plan` Phase 0 to load it | Currently only 2/14 skills read it; widens v1.8.0 lifecycle-split payoff |
| SYSTEM_CONTEXT mtime check | Extract into shared reference at `references/system-context-staleness-check.md`; reference from all 10 consuming skills | DRY; today only `prior-art-research` Phase 0 enforces the check |

## Trade-offs accepted

- **Per-slice review overhead doubles** for slices 1+2 (every reviewer sees changes to all 14 SKILL.md). Counterbalanced by mechanical nature of the changes — diff is regular.
- **Slice 3 (`verify-output`)** is the only new skill in v1.9.0 — adds 1 to the always-loaded description budget. Net token change still negative (slice 1 saves more than slice 3 adds).
- **Slice 4** keeps `/abort-chain` in `using-habeebs-skill` instead of a dedicated skill. Trade-off: not auto-triggered by user phrasing like "cancel" or "abort" — user must invoke explicitly. Acceptable because abort is rare and the cost of a dedicated skill (always-loaded description) outweighs the trigger convenience.
- **GLOSSARY.md consumers stay at 5 of 14 after slice 5** (deep-modules, setup, draft-spec, socratic-grill, write-plan). Skills that don't manipulate domain vocabulary (using-worktrees, parallel-dev, etc.) are correctly excluded.
- **`next-skills` removal is one-way** — if Anthropic later formalizes a chain-frontmatter field, we'd re-add. Low risk; pattern is custom anyway.

### Slice 3 — New `verify-output` skill (AFK)

**Description:** Create a new skill at `skills/verify-output/SKILL.md` plus `skills/verify-output/references/slop-heuristics.md`. The skill runs between `tdd-loop` GREEN (tests passing) and the commit step. It scans the staged diff for slop heuristics from `CLAUDE.md`: unjustified comments, unused variables/imports, defensive error handling on impossible cases, backward-compat shims for code that hasn't shipped, half-finished implementations, premature abstractions (Ousterhout's deletion test if `deep-modules` is loaded). Returns ADR-0004 4-status: `DONE` (clean), `DONE_WITH_CONCERNS` (slop found, listed), `BLOCKED` (slop severe enough to halt commit), `NEEDS_CONTEXT` (can't tell — need user input). Default mode: ANNOTATE (`DONE_WITH_CONCERNS` is the cap; only severe slop blocks). User can override to GATE mode for stricter projects.

**Acceptance criteria:**
- [ ] `skills/verify-output/SKILL.md` exists with frontmatter (name, description ≤ 600 chars, pushy-trigger pattern) and body ≤ 500 lines
- [ ] `skills/verify-output/references/slop-heuristics.md` lifts **4 CLAUDE.md rules verbatim** ("Don't add features…", "Don't add error handling for impossible scenarios", "Default to writing no comments", "Avoid backwards-compatibility hacks") + **3 OMC-paraphrased heuristics** (≤10 words quoted per source): repeated boilerplate suggesting a missed abstraction; defensive validation past trusted boundaries; backward-compat shims for unshipped code. Each heuristic carries a positive example + counter-example.
- [ ] Skill emits one of 4 statuses (`DONE`, `DONE_WITH_CONCERNS`, `BLOCKED`, `NEEDS_CONTEXT`) per ADR-0004
- [ ] **Default mode is ANNOTATE.** `BLOCKED` returns only on **severe slop**: half-finished implementations (functions containing `TODO`, `pass`, or `throw new NotImplementedError`), unreachable code, or imports/variables declared but never used in the staged diff. Moderate slop (unjustified comments, defensive validation) returns `DONE_WITH_CONCERNS` only — never blocks.
- [ ] **GATE mode is opt-in** via skill arg `verify-output --gate`. Documented but not the default.
- [ ] **Does NOT invoke `deep-modules` internally** — separate concern. `deep-modules` continues to fire at `tdd-loop`'s refactor step.
- [ ] **`agent-factors-check` invoked against `verify-output`'s design before commit.** Resulting Socratic questions folded into `slop-heuristics.md` and the SKILL.md pause/resume protocol (especially F6 pause/resume + F7 human-as-tool for ambiguous slop cases).
- [ ] `tdd-loop/SKILL.md` Phase "Refactor and review" updated to invoke `verify-output` between two-stage review and commit
- [ ] Dogfood scenario at `tests/dogfood/12-verify-output/` — 11a planted-moderate-slop (must return `DONE_WITH_CONCERNS`), 11b clean control (must return `DONE`), 11c severe-slop (must return `DONE_WITH_CONCERNS` in default mode, `BLOCKED` in GATE mode)
- [ ] CHANGELOG entry under `## Unreleased` + README skill list updated

**Test strategy:** Dogfood scenarios at `tests/dogfood/12-verify-output/` with 3 fixtures. Plus integration: run the actual `tdd-loop` chain on a small change and confirm `verify-output` is invoked between GREEN and commit.

**Blocked by:** #2 (frontmatter conventions for the new skill must match the post-slice-2 standard — no `next-skills`, trimmed description, `## See also` body section)

**Notes:** Largest new artifact in v1.9.0. Should land before slices 4-5-6 only because it adds a frontmatter consumer of the post-#2 standard; otherwise parallelizable with them.

### Slice 4 — `/abort-chain` convention in `using-habeebs-skill` (AFK)

**Description:** Add a new `## Aborting the chain` section to `skills/using-habeebs-skill/SKILL.md`. Documents: (a) when to abort (user says "stop", "abort", "cancel" in the middle of a chain; new requirement invalidates the active spec; research surfaces a blocker; ADR-0002 violation discovered), (b) cleanup steps in order: flush `docs/agents/SYSTEM_CONTEXT.md` `## Active steering` block, abandon in-flight worktrees per `using-worktrees` Phase 6 teardown, optionally archive a partial spec to `docs/agents/specs/abandoned/`, (c) handoff to either `prior-art-research` (re-research) or back to user (genuinely stuck). NOT a separate skill — lightweight convention.

**Acceptance criteria:**
- [ ] `skills/using-habeebs-skill/SKILL.md` has a new top-level `## Aborting the chain` section
- [ ] Section lists 4+ abort triggers, an **ordered 5-step cleanup checklist**, and 2 handoff paths
- [ ] **5-step cleanup checklist (in order):**
  1. Flush `docs/agents/SYSTEM_CONTEXT.md` `## Active steering` block; copy to `## Last reconciliation outcome` with `(chain aborted YYYY-MM-DD — reason)` header.
  2. List in-flight worktrees via `git worktree list`; prompt user per worktree "Abandon worktree X? (y/n)"; on yes, run `using-worktrees` Phase 6 teardown — `--force` only on second confirmation.
  3. If a partial spec in `docs/agents/specs/` was started after chain start: move to `docs/agents/specs/abandoned/<name>.md` with prepended `**ABANDONED:** YYYY-MM-DD — <reason>` header.
  4. If a partial ADR exists in `docs/agents/adrs/` with status `Draft`: same archive move to `docs/agents/adrs/abandoned/`.
  5. Echo a final summary listing what was cleaned + what was preserved.
- [ ] **No-destructive-git rule:** cleanup performs NO force-pushes, NO branch deletions, NO `git reset --hard`. Worktree removal is the only git op, and only with user confirmation.
- [ ] Cross-referenced from `using-worktrees/SKILL.md` Phase 6 teardown (link to the abort section)
- [ ] CHANGELOG entry under `## Unreleased` documents the new convention
- [ ] Manual smoke test: read the section cold and ask "could an agent execute this without prior context?" — must pass

**Test strategy:** Manual smoke read + grep that the section exists. Lightweight slice — no dogfood needed.

**Blocked by:** #2 (touches `using-habeebs-skill` frontmatter via #2's `next-skills` removal; sequencing avoids merge churn)

**Notes:** Smallest slice. Should be 30-60 min for a focused session.

### Slice 6 — Extract SYSTEM_CONTEXT mtime-check helper + retrofit 10 consumers (AFK)

**Description:** Lift the SYSTEM_CONTEXT.md staleness-check protocol out of `prior-art-research` Phase 0 into a new shared reference file at `docs/agents/references/system-context-staleness-check.md`. The protocol: read SYSTEM_CONTEXT.md mtime, compare against `git log --since "<file_mtime>" -- <manifest_paths>`, emit single banner if stale (`⚠ SYSTEM_CONTEXT.md is stale (X changed since YYYY-MM-DD). Refresh? (Y/n)`), never overwrite silently. Retrofit all 10 SKILL.md files that read SYSTEM_CONTEXT.md to invoke the shared protocol from this reference file rather than re-implementing or trusting.

**Acceptance criteria:**
- [ ] **New file at `docs/agents/references/system-context-staleness-check.md`** (path pinned by grill) exists with the canonical protocol
- [ ] **Failure-mode fallbacks documented in the new reference doc:**
  - If `git log --since` returns non-zero or empty unexpectedly (shallow clone, missing manifests, non-git context): emit advisory `⚠ Could not verify SYSTEM_CONTEXT.md freshness (git history unavailable). Proceeding — context may be stale.` and load file as-is.
  - If `docs/agents/SYSTEM_CONTEXT.md` is missing entirely: emit advisory `⚠ docs/agents/SYSTEM_CONTEXT.md not found. Proceeding with empty cache.` and continue with empty cache (do NOT halt the chain).
- [ ] `prior-art-research/SKILL.md` Phase 0 references the new file (extracts canonical text out of inline definition)
- [ ] 9 other consumers (`agent-factors-check`, `decision-record`, `draft-spec`, `parallel-dev`, `setup-habeebs-skill`, `socratic-grill`, `tdd-loop`, `using-habeebs-skill`, `write-plan`) reference the file in their Phase 0 / context-load step
- [ ] No SKILL.md re-implements the mtime check inline (single source of truth)
- [ ] Test: `grep -l "system-context-staleness-check" skills/*/SKILL.md` returns all 10
- [ ] CHANGELOG entry under `## Unreleased` + brief explanation that this is a DRY refactor, no behavioral change (apart from documented fallbacks)

**Test strategy:** Grep test (all 10 files reference the shared doc). Plus inspection: confirm the shared doc captures the exact protocol from `prior-art-research` Phase 0 verbatim — extraction not paraphrase. Plus a fallback-fixture smoke test: run a skill in a shallow-cloned repo and confirm the advisory appears without halting.

**Blocked by:** #2 (touches 10 SKILL.md files; sequencing avoids merge churn with #2)

**Notes:** The shared reference at `docs/agents/references/system-context-staleness-check.md` introduces the **`docs/agents/references/` directory convention** (resolved by grill — see Architecture section). ## Release

Version bump is **single-shot at the end** (resolved by grill, item 11). Per-slice releases are explicitly rejected.

Procedure once all 6 slices have merged to `main`:

1. Rename CHANGELOG.md `## Unreleased` header to `## v1.9.0 — YYYY-MM-DD`.
2. Bump `version` in `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`, and any other manifest from `1.8.0` to `1.9.0` in a single release commit on `main` titled `v1.9.0: ecosystem alignment + chain hygiene`.
3. Tag: `git tag v1.9.0 && git push --tags`.
4. Create GitHub release: `gh release create v1.9.0` with auto-generated notes plus a short paragraph summarizing the 6 slices.
5. Update `docs/agents/SYSTEM_CONTEXT.md` Methodology section to note `Latest ADR: ADR-0009` (or whichever lands last) and bump the "Last refreshed" date.

Each slice between now and release lands as its own PR; CHANGELOG entries accrue under `## Unreleased`. No `1.9.0-alpha`/`1.9.0-rc` intermediate tags.

## Revisit triggers

Scale or capability conditions that mean the spec needs to be re-evaluated:

- **Anthropic publishes a v2 Skills spec** with `next-skills` (or equivalent chain-frontmatter field): revisit slice 2's removal — re-add formalized version
- **Skill count grows past 20**: re-run description-budget audit (1% of context approaches 5%)
- **User reports `verify-output` blocks too aggressively**: revisit slice 3's GATE vs ANNOTATE default
- **Cross-session observation memory demand surfaces** (users repeatedly hit "what was the error last time"): revisit ADR-0002 for a bounded markdown observation log
- **`/abort-chain` is invoked frequently enough** (>1/week per active user) to justify dedicated skill: promote slice 4's convention to a real skill with auto-triggering
- **GLOSSARY-skip rate exceeds 1-in-5 in dogfood evals**: escalate slice 5's LINK form to LOAD form in v1.10 (consuming skills auto-include GLOSSARY contents in Phase 0 context instead of linking)
- **`docs/agents/references/` grows past ~5 files**: revisit whether sub-categorization is needed (today every entry sits flat)
