# Codex parity test suite

Automated assertions that habeebs-skill is first-class under **Codex CLI** as
well as Claude Code (dual-native parity ADR, `adrs/2026-06-25-dual-native-claude-codex-parity.md`).

These run as part of `tests/run-all.sh` (auto-discovered: `check-*.sh`).

| Scenario | Asserts |
|---|---|
| `01-frontmatter-parity/` | Every `SKILL.md` uses only the portable frontmatter allow-list `{name, description, disable-model-invocation}`, and `name` matches its directory (Codex hard requirement). Guards the minimal-frontmatter decision. |
| `02-skill-drift/` | `.agents/skills/` (the Codex discovery tree) is an in-sync, deterministic mirror of canonical `skills/`, generated by `bin/sync-codex.sh`. |
| `03-codex-hooks/` | `.codex/config.toml` registers the same hook scripts as `hooks/hooks.json`, with matchers valid in Codex's regex dialect. |
| `04-subagent-parity/` | `prior-art-research`, `parallel-dev`, and the `agents/` prompts describe harness-neutral subagent dispatch (Claude Task tool AND Codex subagents). |
